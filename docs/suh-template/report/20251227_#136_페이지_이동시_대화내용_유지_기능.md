# 구현 보고서: 페이지 이동 시 대화내용 유지 기능

**이슈**: #136 - ⚙️ [기능추가][ChatBot] 페이지 이동시에 전체적인 대화내용이 사라지는 문제 개선 필요
**작성일**: 2025-12-27
**담당자**: @Cassiiopeia

---

### 📌 작업 개요

스프링 프레임워크 기반 웹 애플리케이션에서 페이지 이동 시 채팅 내역이 사라지는 문제 해결. localStorage를 활용하여 대화 내용을 저장하고, 페이지 새로고침이나 이동 후에도 대화 히스토리 복원 가능하도록 구현.

추가로 Thinking 프로세스 히스토리 표시 기능 개선 및 LLM 파싱 실패 시 null-safe 처리 추가.

---

### 🎯 구현 목표

1. **대화 내용 영속성**: 페이지 이동/새로고침 후에도 대화 내용 유지
2. **세션 토큰 유지**: localStorage에 세션 토큰 저장하여 서버와의 세션 연속성 확보
3. **Thinking 히스토리 표시**: AI 사고 과정을 히스토리 형태로 상세 표시
4. **안정성 강화**: LLM 응답 파싱 실패 시에도 정상 동작하도록 폴백 로직 추가

---

### ✅ 구현 내용

#### 1. 대화 히스토리 저장/복원 기능 (chatbot.js)
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**:
  - `saveHistory()`: 현재 대화 내용을 localStorage에 JSON 형태로 저장
  - `restoreHistory()`: 페이지 로드 시 저장된 대화 내용 복원
  - `appendMessageFromHistory()`: 히스토리에서 메시지 복원 (저장 트리거 없이)
  - `resetChat()`: 새 대화 시작 시 localStorage 클리어
- **이유**: 스프링 프레임워크의 페이지 기반 렌더링 특성상 JavaScript 상태가 초기화되므로, localStorage를 통한 영속성 확보 필요

#### 2. Thinking 히스토리 표시 기능 (chatbot.js)
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**:
  - `thinkingHistoryMap`: thinking 이벤트 히스토리 저장소 추가 (messageId -> [events])
  - `handleThinkingEvent()`: 히스토리 형태로 thinking 이벤트 누적 렌더링
  - `buildThinkingMessage()`: 사용자 친화적 메시지 생성
  - `getThinkingIcon()`: 단계별 아이콘 반환 (스피너, 체크, 스킵 등)
  - `getThinkingStatusClass()`: 상태별 CSS 클래스 반환
  - `truncateText()`: UI용 텍스트 자르기 유틸리티
- **이유**: 기존에는 "생각하고 있어요...", "답변을 작성하고 있어요..." 두 가지만 표시되었으나, 구체적인 AI 사고 과정을 사용자가 볼 수 있도록 개선

#### 3. Thinking 상세 정보 전송 (ChatbotService.java)
- **파일**: `Suh-Domain-Chatbot/src/main/java/me/suhsaechan/chatbot/service/ChatbotService.java`
- **변경 내용**:
  - Step 1: 의도 분류 결과 + 신뢰도 퍼센트 표시 (예: "지식 질문 (95%)")
  - Step 2: 검색 결과 + 최고 유사도 표시 (예: "3개 문서 (최고 유사도 87%)")
  - Step 3: 컨텍스트 정보 표시 (예: "검색된 3개 문서 기반 답변 작성 중...")
- **이유**: 프론트엔드에서 상세한 thinking 히스토리를 표시할 수 있도록 백엔드에서 구체적인 정보 전달

#### 4. LLM 파싱 실패 시 Null-safe 처리 (ChatbotService.java)
- **파일**: `Suh-Domain-Chatbot/src/main/java/me/suhsaechan/chatbot/service/ChatbotService.java`
- **변경 내용**:
  - `isValidIntentResult()`: 의도 분류 결과 유효성 검증 (intentType, needsRagSearch, confidence null 체크)
  - `createFallbackIntent()`: 파싱 실패 시 기본 의도 분류 결과 생성
    - `intentType`: "KNOWLEDGE_QUERY"
    - `needsRagSearch`: true (안전을 위해 RAG 검색 수행)
    - `confidence`: 0.5f
- **이유**: LLM이 예상 형식과 다른 JSON을 반환할 경우 NullPointerException 발생 방지

#### 5. JSON 역직렬화 오류 방지 (IntentClassificationDto.java)
- **파일**: `Suh-Domain-Chatbot/src/main/java/me/suhsaechan/chatbot/dto/IntentClassificationDto.java`
- **변경 내용**: `@JsonIgnoreProperties(ignoreUnknown = true)` 어노테이션 추가
- **이유**: LLM이 예상치 못한 필드(예: "type")를 반환해도 Jackson 역직렬화 오류 없이 처리

### 🔧 주요 변경사항 상세

#### chatbot.js - 히스토리 저장/복원

**저장 구조**:
```javascript
{
  sessionToken: "uuid-string",
  messages: [
    { role: "user", content: "메시지 내용", time: "오후 8:30" },
    { role: "assistant", content: "응답 내용", time: "오후 8:31" }
  ]
}
```

**초기화 시점**:
- `init()`: 페이지 로드 시 sessionToken 복원 및 히스토리 복원
- `resetChat()`: 새 대화 시작 시 localStorage 클리어

**저장 시점**:
- `appendMessage()`: 새 메시지 추가 시 자동 저장
- `finalizeStreamingMessage()`: 스트리밍 완료 시 저장

#### chatbot.js - Thinking 히스토리

**Step별 메시지 포맷**:
- Step 1 (질문 분석): "질문을 분석하고 있어요..." → "지식 질문 (95%) 질문이네요!"
- Step 2 (문서 검색): ""SUH Project Utility..." 관련 문서를 찾고 있어요..." → "3개 문서 (최고 유사도 87%)"
- Step 3 (응답 생성): "검색된 3개 문서 기반 답변 작성 중..." → "답변 준비 완료!"

**상태별 아이콘**:
- `in_progress` / `retrying`: 스피너 (`fa-spinner fa-spin`)
- `completed`: 체크 아이콘 (`fa-check-circle`)
- `skipped`: 스킵 아이콘 (`fa-forward`)

#### ChatbotService.java - Null-safe 처리

**폴백 로직 흐름**:
1. `classifyUserIntentInternal()` 호출하여 1차 의도 분류
2. `isValidIntentResult()`로 결과 유효성 검증
3. 유효하지 않으면 `createFallbackIntent()` 호출하여 기본값 반환
4. 유효하면 신뢰도 검사 후 필요시 재시도

**기본값 설정**:
- 안전을 위해 `needsRagSearch: true`로 설정하여 RAG 검색 수행
- `confidence: 0.5f`로 설정하여 낮은 신뢰도 표시

### 🧪 테스트 및 검증

1. **대화 히스토리 유지 테스트**
   - 채팅 후 페이지 새로고침 → 대화 내용 유지 확인
   - 다른 페이지로 이동 후 복귀 → 대화 내용 유지 확인
   - "새 대화" 버튼 클릭 → 대화 내용 초기화 확인

2. **Thinking 히스토리 표시 테스트**
   - 질문 입력 후 3단계 thinking 과정이 히스토리 형태로 표시되는지 확인
   - 각 단계별 상세 정보(신뢰도, 유사도 등) 표시 확인

3. **LLM 파싱 실패 시 폴백 테스트**
   - LLM이 잘못된 형식 반환 시에도 NullPointerException 없이 정상 동작 확인
   - 기본값(KNOWLEDGE_QUERY)으로 폴백 후 RAG 검색 및 응답 생성 확인

### 📌 참고사항

- **서버 재시작 필요**: ChatbotService.java 수정사항 적용을 위해 서버 재시작 필요
- **localStorage 용량**: 대화 히스토리가 길어지면 localStorage 용량 제한(5MB)에 도달할 수 있음. 향후 오래된 메시지 자동 삭제 로직 고려 필요
- **세션 만료**: 서버 측 세션이 만료되어도 클라이언트 측 히스토리는 유지됨. 이 경우 새 세션으로 대화 이어가기 가능
