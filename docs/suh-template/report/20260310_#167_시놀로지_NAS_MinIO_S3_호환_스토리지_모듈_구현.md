### 📌 작업 개요

시놀로지 NAS에 MinIO S3 호환 스토리지 서버 인프라 구축 및 Spring Boot 연동 코드 구현. Docker Compose로 MinIO 컨테이너 배포, 역방향 프록시(콘솔 + S3 API) 설정, DNS 연결, 버킷 생성 및 접근 정책 설정, Spring Boot MinIO SDK 연동 유틸리티까지 완료.

### 🎯 구현 목표

- 시놀로지 NAS를 S3 호환 오브젝트 스토리지 서버로 활용
- MinIO SDK를 통한 파일 업로드/다운로드/삭제/Presigned URL 기능 제공
- 버킷 내 `public/` 경로는 외부 직접 접근 허용, `private/` 경로는 인증 필요한 구조 구현
- 기존 SMB/SSH 방식의 파일 접근 제약 해소 (공개/비공개 제어, 버킷 단위 관리)

### ✅ 구현 내용

#### 1. MinIO Docker Compose 배포
- **파일**: `docker-compose.yml` (신규)
- **이미지**: `minio/minio:RELEASE.2025-04-22T22-12-26Z` (GUI 관리 기능 포함 마지막 버전)
- **포트**: API `9000`, 웹 콘솔 `9090`
- **마운트**: `/volume1/s3:/data`
- **환경변수**: `MINIO_BROWSER_REDIRECT_URL`, `MINIO_SERVER_URL` 설정

**특이사항**:
- 2025년 5월 이후 MinIO 커뮤니티 에디션에서 Access Policy UI, 사용자 관리, 버킷 관리 등 GUI 기능 제거됨 (상용 AIStor로 전환)
- GUI 관리 기능이 필요하여 `RELEASE.2025-04-22T22-12-26Z` 버전으로 고정

#### 2. 시놀로지 역방향 프록시 설정 (4개 규칙)
- `minio.suhsaechan.kr:80` → HTTPS 리다이렉트
- `minio.suhsaechan.kr:443` → `localhost:9090` (WebSocket 활성화) — 웹 콘솔
- `s3.suhsaechan.kr:80` → HTTPS 리다이렉트
- `s3.suhsaechan.kr:443` → `localhost:9000` (WebSocket 활성화) — S3 API

#### 3. Cloudflare DNS 설정
- CNAME `minio` → `suh-project.synology.me` (Proxied)
- CNAME `s3` → `suh-project.synology.me` (Proxied)

#### 4. 버킷 생성 및 접근 정책
- `suh-project-utility` 버킷 — 현재 프로젝트용
- `romrom` 버킷 — RomRom 프로젝트용
- 각 버킷 Custom Access Policy 적용: `public/*` 경로만 익명 읽기 허용

**정책 파일**:
- `minio-policy/suh-project-utility-policy.json` (신규)
- `minio-policy/romrom-policy.json` (신규)

#### 5. Spring Boot MinIO 연동 - Properties
- **파일**: `Suh-Common/.../properties/MinioProperties.java` (신규)
- `@ConfigurationProperties(prefix = "minio")` 패턴 (기존 `SshConnectionProperties`와 동일)
- 필드: `endpoint`, `accessKey`, `secretKey`, `bucket`

#### 6. Spring Boot MinIO 연동 - 유틸리티
- **파일**: `Suh-Common/.../util/MinioUtil.java` (신규)
- `@Component` + `@RequiredArgsConstructor` 생성자 주입
- `@PostConstruct`로 `MinioClient` 초기화

제공 메서드:

| 카테고리 | 메서드 | 설명 |
|---------|--------|------|
| 버킷 | `bucketExists` | 버킷 존재 확인 |
| 버킷 | `createBucket` | 버킷 생성 (이미 존재 시 스킵) |
| 버킷 | `removeBucket` | 버킷 삭제 |
| 버킷 | `listBuckets` | 전체 버킷 목록 조회 |
| 업로드 | `uploadFile` (InputStream) | 스트림 기반 파일 업로드 |
| 업로드 | `uploadFile` (MultipartFile) | Spring MultipartFile 업로드 |
| 다운로드 | `downloadFile` | InputStream 반환 |
| 다운로드 | `downloadFileAsBytes` | byte[] 반환 (InputStream 자동 close) |
| 삭제 | `deleteFile` | 오브젝트 삭제 |
| 조회 | `getFileInfo` | 파일 메타정보 (크기, 타입, ETAG) |
| 조회 | `listObjects` | 오브젝트 목록 (prefix 필터, recursive 지원) |
| URL | `getPresignedDownloadUrl` | 임시 다운로드 링크 (기본 1시간) |
| URL | `getPresignedUploadUrl` | 임시 업로드 링크 (기본 1시간) |

경로 상수 제공:

| 상수 | 값 | 용도 |
|------|---|------|
| `PUBLIC_PATH` | `public/` | 외부 접근 가능 일반 파일 |
| `PUBLIC_IMAGES_PATH` | `public/images/` | 외부 접근 가능 이미지 |
| `PRIVATE_PATH` | `private/` | 인증 필요 비공개 파일 |

### 📌 참고사항

- MinIO 버전 `RELEASE.2025-04-22T22-12-26Z`로 고정 (GUI 관리 기능 포함 마지막 버전)
- `MINIO_BROWSER_REDIRECT_URL`과 `MINIO_SERVER_URL` 환경변수는 역방향 프록시 설정과 반드시 매칭 필요 (불일치 시 503 에러 발생)
- `MINIO_SERVER_URL`은 Docker 컨테이너 내부에서 접근 가능한 주소여야 함 → `http://localhost:9000` 사용 (외부 도메인 사용 시 DNS 해석 불가)
- 역방향 프록시에서 WebSocket 활성화 필수 (미활성 시 Object Browser 무한 로딩)
- 이미지 외부 공개: 버킷 Access Policy를 Custom으로 설정, `public/*` 경로에 `s3:GetObject` 허용
- 공개 이미지 직접 접근: `https://s3.suhsaechan.kr/{버킷명}/public/images/{파일명}`
- 비공개 파일은 Presigned URL로 임시 공유 가능 (기본 만료 시간 1시간)
