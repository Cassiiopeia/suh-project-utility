### 📌 작업 개요
GitHub Actions 워크플로우에서 "자동 PR Merge" 단계 실행 시 `GH_TOKEN` 환경 변수 미설정으로 인한 오류 수정

**보고서 파일**: `.report/20251214_워크플로우_GH_TOKEN_오류수정.md`

### 🔍 문제 분석
워크플로우는 정상적으로 작동하지만, "자동 PR Merge" 단계에서 `gh pr view` 명령 실행 시 `GH_TOKEN` 환경 변수가 설정되지 않아 다음 오류 발생:

```
gh: To use GitHub CLI in a GitHub Actions workflow, set the GH_TOKEN environment variable.
##[error]Process completed with exit code 4.
```

**원인**:
- "PR 브랜치 최신화" 단계(275-318줄)에서는 `env:` 블록으로 `GH_TOKEN`을 설정하여 정상 작동
- "자동 PR Merge" 단계(320-465줄)에서는 `env:` 블록이 없어서 `gh` 명령 실행 시 환경 변수 미설정
- 스크립트 내에서 `export GH_TOKEN=...`로 설정하지만, 334줄의 `gh pr view` 명령이 실행되는 시점에는 아직 설정되지 않음

### ✅ 구현 내용

#### "자동 PR Merge" 단계에 env 블록 추가
- **파일**: `.github/workflows/PROJECT-COMMON-AUTO-CHANGELOG-CONTROL.yaml`
- **변경 내용**: 320-323줄에 `env:` 블록 추가하여 `GH_TOKEN` 환경 변수를 기본값으로 설정
- **이유**: 모든 `gh` 명령 실행 전에 환경 변수가 설정되도록 보장

### 🔧 주요 변경사항 상세

#### 변경 전
```yaml
- name: 자동 PR Merge
  run: |
    PR_NUMBER=${{ github.event.pull_request.number }}
    
    # PAT 토큰 존재 여부 확인
    ...
    
    # PR 상태 확인
    PR_STATE=$(gh pr view $PR_NUMBER --json state,mergeable -q '.state + "," + (.mergeable | tostring)')
    # ↑ 여기서 GH_TOKEN이 설정되지 않아 오류 발생
```

#### 변경 후
```yaml
- name: 자동 PR Merge
  env:
    GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
  run: |
    PR_NUMBER=${{ github.event.pull_request.number }}
    
    # PAT 토큰 존재 여부 확인
    ...
    
    # PR 상태 확인
    PR_STATE=$(gh pr view $PR_NUMBER --json state,mergeable -q '.state + "," + (.mergeable | tostring)')
    # ↑ 이제 GH_TOKEN이 설정되어 정상 작동
```

**특이사항**:
- `env:` 블록은 step 전체에 적용되어 모든 `gh` 명령 실행 전에 환경 변수 설정됨
- 스크립트 내의 `export GH_TOKEN=...` 부분은 명시적 재설정용으로 그대로 유지 (PAT 토큰 폴백 로직 보존)
- "PR 브랜치 최신화" 단계와 동일한 패턴 적용하여 일관성 유지

### 🧪 테스트 및 검증
- YAML 문법 검증: 린터 오류 없음 확인
- 기존 로직 보존: PAT 토큰 폴백 로직이 정상 작동하는지 확인
- 환경 변수 설정: 모든 `gh` 명령 실행 전에 `GH_TOKEN`이 설정되는지 확인

### 📌 참고사항
- 워크플로우 실행 시 오류 없이 정상 작동해야 함
- `gh pr view`, `gh pr merge`, `gh pr comment` 등 모든 `gh` 명령이 정상 실행됨
- 필요시 PAT 토큰으로 폴백하는 기존 로직도 그대로 유지됨
