## 📌 작업 개요

로그인 페이지에서 아이디 또는 비밀번호 입력 필드 클릭 시 두 입력 필드가 서로 붙는 CSS 레이아웃 문제 수정. Tailwind 유틸리티 클래스와 커스텀 CSS의 충돌을 최소화하여 해결.

**보고서 파일**: `.report/20260115_#150_로그인페이지_입력필드_클릭시_간격붙는_현상_수정.md`

## 🔍 문제 분석

### 문제 상황
- 로그인 페이지 최초 접근 시에는 정상적으로 표시
- 아이디 또는 비밀번호 입력 필드 클릭 시 두 입력 필드가 서로 붙어버림
- 입력 필드 간 간격이 0이 되어 시각적으로 혼란스러운 UI

### 원인 분석
**원인 1: `!mt-0` 강제 오버라이드**
- Password 입력 필드의 `form-control`에 `!mt-0` 클래스 적용
- Tailwind의 `space-y-4`를 `!important`로 무효화
- 기본 상태에서 이미 Username과 붙어있는 상태

**원인 2: `input:focus` 테두리 크기 증가**
```css
/* 기존 코드 - 문제 발생 */
input:focus {
  border: 2px solid #3b82f6 !important;  /* 1px → 2px 증가 */
}
```
- DaisyUI 기본 input 테두리: 1px
- focus 시 테두리: 2px로 증가
- 테두리 크기 증가로 인한 레이아웃 shift 발생
- box-shadow 추가로 추가 공간 점유

**원인 3: validator-hint 동적 표시**
- `validator-hint`가 `display: none` → `display: block` 전환
- 공간 차지 변화로 아래 컴포넌트가 밀림
- `!mt-0` 때문에 Password 필드가 위로 당겨지며 붙음

## ✅ 구현 내용

### 1. HTML 구조 단순화
- **파일**: `Suh-Web/src/main/resources/templates/pages/login.html` (63줄)
- **변경 내용**: Password 필드의 `!mt-0` 클래스 제거
- **이유**: Tailwind의 `space-y-4` 유틸리티가 정상 작동하도록 하여 일정한 간격 유지

```html
<!-- 변경 전 -->
<div class="form-control !mt-0">

<!-- 변경 후 -->
<div class="form-control">
```

### 2. CSS focus 스타일 최소화
- **파일**: `Suh-Web/src/main/resources/static/css/common.css` (1159줄)
- **변경 내용**: `border: 2px` → `border-color`로 변경
- **이유**: 테두리 크기 변경을 제거하여 레이아웃 shift 방지. DaisyUI 기본 1px 테두리 크기 유지하며 색상만 파란색으로 변경

```css
/* 변경 전 - 레이아웃 shift 발생 */
input:focus, textarea:focus, select:focus {
  outline: none;
  border: 2px solid #3b82f6 !important;
  box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.1);
}

/* 변경 후 - 크기 유지, 색상만 변경 */
input:focus, textarea:focus, select:focus {
  outline: none;
  border-color: #3b82f6 !important;
  box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.1);
}
```

### 3. 다크모드 스타일 동기화
- **파일**: `Suh-Web/src/main/resources/static/css/common.css` (2810줄)
- **변경 내용**: 다크모드에서도 동일하게 `border-color`만 변경
- **이유**: 라이트모드와 동일한 동작 보장

```css
/* 변경 후 */
[data-theme="dark"] .chatbot-input-area textarea:focus {
  border-color: #3b82f6 !important;
  box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.2) !important;
}
```

## 🔧 주요 변경사항 상세

### Tailwind 유틸리티 클래스 정상화
`!mt-0` 제거로 `space-y-4`가 정상 작동. Username과 Password 필드 사이에 일정한 간격(1rem = 16px) 자동 확보.

**특이사항**:
- 간격을 확대하지 않고 기존 `space-y-4` 유지
- Tailwind 유틸리티만으로 간격 관리하여 커스텀 CSS 최소화

### CSS 오버라이드 최소화
`border: 2px` → `border-color`로 변경하여 DaisyUI 기본 테두리 크기(1px) 유지. focus 시 색상만 파란색으로 변경하여 시각적 피드백 제공하며 레이아웃 변화는 방지.

**특이사항**:
- 테두리 크기 변경 제거로 레이아웃 shift 완전 방지
- box-shadow는 유지하여 focus 시 부드러운 시각적 효과 제공
- 다른 페이지의 input 필드에도 동일한 일관성 적용
