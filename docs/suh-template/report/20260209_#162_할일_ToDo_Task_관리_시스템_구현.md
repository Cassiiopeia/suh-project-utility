### 📌 작업 개요
문제집 풀이 등 개인 목표를 Task 단위로 관리하고 진행률을 추적하는 Task 트래커 시스템 구현. 대시보드에 D-Day 카드로 표시되며, 별도 관리 페이지에서 목표 생성/수정/삭제 및 일별 진행 기록 관리 가능

**보고서 파일**: `.report/20260209_#162_할일_ToDo_Task_관리_시스템_구현.md`

### 🎯 구현 목표
- Task 목표(Goal) 생성 및 관리 기능
- 일별 진행 기록(Progress) 추적 기능
- 대시보드에 D-Day 카드로 목표 현황 표시
- 진행률 시각화 (프로그레스 바, 범위 슬라이더)

### ✅ 구현 내용

#### 신규 도메인 모듈 생성 (Suh-Domain-Task)
- **파일**: `Suh-Domain-Task/` 전체
- **변경 내용**: Task 도메인 모듈 신규 생성. Entity, Repository, Service, DTO 구조 구성
- **이유**: 기존 도메인 모듈 패턴과 동일한 구조로 독립 모듈 분리

#### TaskGoal 엔티티 구현
- **파일**: `Suh-Domain-Task/src/main/java/me/suhsaechan/task/entity/TaskGoal.java`
- **변경 내용**: 목표 관리 엔티티 구현. title, description, targetDate, totalAmount, unit(기본값: "페이지"), icon, color, isActive, isCompleted, priority 필드 포함
- **이유**: 문제집 풀이 등 다양한 목표를 유연하게 관리하기 위한 범용 구조 설계

#### TaskProgress 엔티티 구현
- **파일**: `Suh-Domain-Task/src/main/java/me/suhsaechan/task/entity/TaskProgress.java`
- **변경 내용**: 일별 진행 기록 엔티티 구현. progressDate, content, startAmount, endAmount, memo, isCompleted 필드 포함. TaskGoal과 @ManyToOne 관계 설정
- **이유**: 매일의 진행 상황을 시작~끝 범위로 기록하여 정확한 진행률 산출

#### Repository 구현
- **파일**: `TaskGoalRepository.java`, `TaskProgressRepository.java`
- **변경 내용**: 활성 목표 조회, 우선순위 정렬, 제목 검색, 날짜 범위 진행 기록 조회, 최대 진행량 조회(@Query JPQL) 등 커스텀 쿼리 메서드 구현
- **이유**: 다양한 필터링과 통계 산출에 필요한 쿼리 지원

#### Service 계층 구현
- **파일**: `TaskGoalService.java`, `TaskProgressService.java`
- **변경 내용**:
  - Goal: 생성, 조회, 수정(null-safe 부분 업데이트), 삭제(Progress 연쇄 삭제), 완료/취소 상태 전환
  - Progress: 저장(동일 날짜 존재 시 업데이트, 새 날짜면 생성), 목표별/전체 조회, 삭제
- **이유**: 멱등성 있는 진행 기록 저장과 연쇄 삭제 등 비즈니스 로직 캡슐화

#### DTO 구조
- **파일**: `TaskRequest.java`, `TaskResponse.java`, `TaskGoalDto.java`, `TaskProgressDto.java`
- **변경 내용**: 단일 Request/Response 패턴 적용. TaskGoalDto에 daysRemaining, currentAmount, progressPercentage 계산 필드 추가. TaskProgressDto에 amountDone 계산 필드 추가
- **이유**: 프로젝트 컨벤션(단일 Request/Response) 준수 및 프론트엔드에서 바로 사용 가능한 계산값 제공

#### API 컨트롤러 구현
- **파일**: `Suh-Web/src/main/java/me/suhsaechan/web/controller/api/TaskController.java`
- **변경 내용**: 10개 API 엔드포인트 구현
  - Goal: `/api/task/goal/create`, `/list`, `/detail`, `/update`, `/delete`, `/complete`, `/cancel`
  - Progress: `/api/task/progress/save`, `/list`, `/delete`
- **이유**: 목표와 진행 기록에 대한 CRUD 및 상태 전환 API 제공

#### Task 트래커 페이지 구현

### 🔧 주요 변경사항 상세

#### 진행률 계산 로직
TaskGoalDto 변환 시 해당 목표의 모든 Progress에서 MAX(endAmount)를 조회하여 currentAmount로 사용. progressPercentage = (currentAmount / totalAmount) * 100으로 산출

#### 멱등성 있는 Progress 저장
동일 목표 + 동일 날짜에 이미 기록이 존재하면 UPDATE, 없으면 CREATE. 하루에 여러 번 저장해도 중복 기록 생성되지 않음

#### 듀얼 레인지 슬라이더
CSS 커스텀 스타일링으로 두 개의 range input을 겹쳐서 시작~끝 범위 선택 가능. 선택 범위를 시각적으로 하이라이트 표시. 다크모드 지원

**특이사항**:
- Goal 삭제 시 연관된 모든 Progress 레코드를 먼저 삭제한 후 Goal 삭제 (CASCADE 미사용, Repository에서 직접 삭제)
- unit 기본값 "페이지"로 설정하여 문제집 풀이 용도에 최적화
