# [기능개선][Study] Suh-Domain-Study 모듈 및 스터디 플랜 UI 제거

## 개요
더 이상 사용하지 않는 Suh-Domain-Study 모듈 전체를 삭제하고, 대시보드 및 헤더에서 스터디 플랜 관련 UI를 제거했다. 총 1,684줄이 삭제되어 프로젝트 의존성과 복잡도가 줄었다.

## 변경 사항

### 모듈 설정
- `settings.gradle`: `include 'Suh-Domain-Study'` 제거.
- `Suh-Application/build.gradle`: Study 모듈 의존성 제거.
- `Suh-Web/build.gradle`: Study 모듈 의존성 제거.

### 컨트롤러 삭제
- `StudyAttachmentController.java`: 첨부파일 업로드/다운로드 API 컨트롤러 전체 삭제.
- `StudyManagementController.java`: 카테고리·포스트 CRUD API 컨트롤러 전체 삭제.
- `StudyViewController.java`: 스터디 뷰 페이지 컨트롤러 전체 삭제.

### UI 템플릿 삭제
- `templates/pages/study/category-management.html`: 카테고리 관리 페이지 삭제.
- `templates/pages/study/category.html`: 카테고리 목록 페이지 삭제.
- `templates/pages/study/post-form.html`: 포스트 작성/수정 폼 삭제.
- `templates/pages/study/post-view.html`: 포스트 상세 뷰 삭제.
- `templates/pages/study/studyManagementMain.html`: 스터디 메인 페이지 삭제.

### UI 수정
- `templates/fragments/header.html`: 헤더의 스터디 플랜 메뉴 링크 제거.
- `templates/pages/dashboard.html`: 대시보드 서비스 카드 섹션에서 "스터디 플랜" 카드 제거.

### 서비스/DTO 수정
- `DashboardService.java`: Study 관련 통계 집계 로직 제거.
- `DashboardSummaryDto.java`: Study 관련 필드 제거.
- `PageVisitInterceptor.java`: Study 페이지 방문 추적 로직 제거.

## 주요 구현 내용
Study 도메인 모듈을 `settings.gradle`에서 제외하고, 이를 참조하던 Application/Web 모듈의 의존성을 함께 제거했다. 연쇄적으로 참조하던 컨트롤러, 뷰 템플릿, 서비스 로직까지 전체 삭제하여 빌드 오류 없이 완전히 제거했다.

## 주의사항
- MinIO 연동(이슈 #167)이 Study 모듈에 적용되어 있었으나, Study 모듈 삭제로 해당 구현도 함께 제거됨. MinIO 자체는 다른 모듈에서 독립적으로 사용 가능.
