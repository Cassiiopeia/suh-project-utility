# 버스 자동예약 월요일 누락 버그 수정

## 개요
소만사 버스 자동예약 스케줄러가 월요일 좌석을 예약하지 못하는 버그를 수정했습니다. 근본 원인은 `computeNextFireAt`에서 발화 날짜를 선택할 때 설정값 `SOMANSA_BUS_SCHEDULER_DAYS`를 잘못 해석했던 것입니다. 설정값을 "예약 대상일의 요일"로 통일하여 일요일 밤에 정상적으로 발화하고 월요일 좌석을 예약할 수 있도록 수정했습니다.

## 변경 사항

### 1. 스케줄러 로직 수정
- **파일**: `Suh-Domain-Somansa-Bus/src/main/java/.../service/SomansaBusSchedulerService.java`
- **변경**: `computeNextFireAt` 메서드의 while 루프 조건 수정
  - **기존**: `candidateDate.getDayOfWeek()` — 발화일 자체의 요일을 검사
  - **수정**: `candidateDate.plusDays(1).getDayOfWeek()` — 발화일의 다음날(예약 대상일)의 요일을 검사
  - **효과**: 발화일이 자동으로 조정되어, 월요일 예약을 위해 일요일 밤에 발화하도록 복구

### 2. 설정값 설명 정정
- **파일**: `Suh-Common/src/main/java/.../constant/ServerOptionKey.java`
- **변경**: `SOMANSA_BUS_SCHEDULER_DAYS` 설명 텍스트 수정
  - **기존**: "자동 예약 실행 요일" — 모호한 표현
  - **수정**: "자동 예약 대상일 요일" + 구체적 예시 추가
  - **목적**: 설정값의 진정한 의미를 명확히 하여 향후 유사 오류 방지

### 3. 테스트 케이스 수정
- **파일**: `Suh-Domain-Somansa-Bus/src/test/java/.../service/SomansaBusSchedulerServiceTest.java`
- **변경**: `computeNextFireAt_허용요일_전진_테스트` 기대값 수정
  - DAYS="MONDAY" 지정 시 발화일이 **SUNDAY**(기존: MONDAY)가 되어야 함을 확인
  - 예약 대상일 = 발화일 + 1 = MONDAY 검증 추가

## 주요 구현 내용

### 문제의 원인
두 함수가 같은 설정값(`SOMANSA_BUS_SCHEDULER_DAYS = 월~금`)을 서로 다르게 해석하고 있었습니다:
- `tick()`: 설정값을 **"예약 대상일의 요일"**로 해석 (내일 요일과 비교)
- `computeNextFireAt()`: 설정값을 **"발화하는 날짜의 요일"**로 해석 (발화일 자체와 비교)

`computeNextFireAt`이 발화일을 월~금으로만 제한하면서 일요일에는 발화 후보가 만들어지지 않았고, 그 결과 월요일 좌석 예약이 영구 누락됐습니다.

### 해결 방식
설정값의 의미를 **"예약 대상일의 요일"**로 통일했습니다. while 루프가 발화일 자체가 아니라 **발화일의 다음날(예약 대상일)이 allowedDays에 속하는지 확인**하도록 변경하면서, 자동으로 올바른 발화일이 선택됩니다:
- 월요일 좌석 예약 대상 → 일요일 발화 (일요일 + 1 = 월요일 ✓)
- 화~금 좌석 예약 대상 → 월~목 발화
- 토/일 좌석은 예약 대상에서 제외

### 검증
요일별 시뮬레이션 결과, 도달 가능한 예약 대상일이 정확히 {월,화,수,목,금}이며, 토/일은 절대 예약 대상이 되지 않음을 확인했습니다.

## 주의사항

- **DB 기존 `nextFireAt` 값**: 의미가 동일하게 유지되므로 값 변경 불필요. 다음 발화 시 자동 재계산됩니다.
- **테스트 실행**: Gradle 빌드는 내부망에서 불가능하므로, 별도 환경에서 `./gradlew :Suh-Domain-Somansa-Bus:test` 실행하여 회귀 확인이 필요합니다.
- **향후 개선**: 설정값 의미를 코드·문서에서 더욱 명확히 하여 같은 오류 재발 방지.
