# [버그][대시보드] 비로그인 상태에서 대시보드 API 로그인 페이지 리다이렉트 문제

## 개요
비로그인 상태에서 대시보드 페이지 로드 시 호출하는 API 3개(`/api/dashboard/summary`, `/api/notice/get/active`, `/api/task/goal/list`)가 인증 필터에 막혀 로그인 페이지로 리다이렉트되는 문제를 수정했다. 또한 이전 작업에서 누락된 대시보드 프로필 섹션(About Me, Timeline, Certifications, Tech Stack, Team/Personal Projects, Nexus Modules)을 복원했다.

## 변경 사항

### 보안 설정
- `Suh-Web/src/main/java/me/suhsaechan/web/config/PublicEndpointConfig.java`: 공개 API 목록에 `/api/dashboard/summary`, `/api/notice/get/active`, `/api/task/goal/list` 3개 엔드포인트 추가.

### UI 템플릿
- `Suh-Web/src/main/resources/templates/pages/dashboard.html`: About Me, Timeline, Certifications, Tech Stack, Team/Personal Projects, Nexus Modules 섹션 복원 (694줄 추가).
- `Suh-Web/src/main/resources/static/css/common.css`: 복원된 섹션에 필요한 스타일 12줄 추가.

## 주요 구현 내용
`PublicEndpointConfig`에서 `List.of()`로 관리되는 공개 허용 URL 목록에 누락된 대시보드 관련 API를 추가했다. 이 설정은 `WebSecurityConfig`에서 참조하여 `permitAll()` 처리된다. 비로그인 사용자도 대시보드의 핵심 콘텐츠(통계, 공지, Task D-Day)를 정상 조회할 수 있게 되었다.

## 주의사항
- 공개 API 목록에 추가된 3개 엔드포인트는 인증 없이 접근 가능하므로, 해당 API가 민감 정보를 반환하지 않는지 사전 확인이 필요하다.
- 프로필 섹션 복원은 이전 리팩토링 작업에서 발생한 누락이므로, 향후 대규모 UI 변경 시 전체 섹션 렌더링 여부를 검증해야 한다.
