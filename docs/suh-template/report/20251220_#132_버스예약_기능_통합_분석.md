## 📌 분석 요약

외부 레포지토리(`auto-bus-reservation`)에서 관리 중이던 버스 예약 기능을 현재 레포(`suh-project-utility`)로 통합하여 관리하도록 변경하는 작업입니다. 핵심 목표는 **ENUM 기반 하드코딩된 설정을 DB 기반 동적 관리로 전환**하고, **프론트엔드에서 예약 스케줄링을 관리할 수 있는 UI를 제공**하는 것입니다.

**감지된 프로젝트 타입**: Spring Boot 3.4.2 멀티 모듈 프로젝트  
**주요 기술 스택**: 
- Backend: Java 17, Spring Boot 3.4.2, Spring Data JPA, Spring Security
- Database: PostgreSQL, Redis
- Frontend: Thymeleaf, Tailwind CSS, JavaScript
- Build: Gradle 8.12.1

**코드 스타일**: 
- 엔티티: `BasePostgresEntity` 상속, `@SuperBuilder`, `@Getter`, `@NoArgsConstructor`, `@AllArgsConstructor`
- 서비스: `@RequiredArgsConstructor` + 생성자 주입, `@Transactional(readOnly = true)` 기본
- 컨트롤러: `@RestController`, `@RequestMapping("/api/{domain}")`, POST 방식, `MediaType.MULTIPART_FORM_DATA_VALUE`
- 페이지: `@Controller`, `@GetMapping`, Thymeleaf 템플릿 (`pages/{pageName}.html`)

- 중복된 설정 클래스 제거 필요

### 3. ENUM 제거 시점
- DB 마이그레이션 완료 후 ENUM 제거
- 기존 코드에서 ENUM 참조하는 부분 모두 수정 필요

### 4. 스케줄링 전환
- 기존 `@Scheduled` 어노테이션 제거
- 동적 스케줄링으로 전환 시 기존 스케줄 중단 없이 전환 필요

### 5. 데이터 일관성
- 초기 데이터 마이그레이션 시 기존 ENUM 데이터와 일치하도록 주의
- 스케줄 활성화/비활성화 시 실행 중인 스케줄에 영향 없도록 처리

### 6. 성능 고려사항
- 동적 스케줄링 시 DB 조회 빈도 최적화
- 예약 결과 조회 시 페이징 처리

### 7. 보안 고려사항
- 버스 예약 API는 인증된 사용자만 접근 가능하도록 설정
- 관리자 권한이 필요한 기능은 권한 체크 추가
