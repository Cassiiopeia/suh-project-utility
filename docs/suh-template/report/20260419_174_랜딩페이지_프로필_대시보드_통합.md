# 랜딩 페이지를 프로필+대시보드 통합 페이지로 변경 및 로그인 최소화

## 개요
기존에 분리되어 있던 `profile.html`을 삭제하고, 프로필 섹션을 `dashboard.html`에 통합했다. `/`(루트)와 `/dashboard`를 비인증 공개 경로로 추가하여 로그인 없이도 대시보드+프로필을 한 페이지에서 볼 수 있도록 변경했다.

## 변경 사항

### 보안 설정
- `PublicEndpointConfig.java`: 공개 리소스 엔드포인트 목록에 `/`와 `/dashboard` 추가. 비로그인 사용자도 메인 페이지 접근 가능.

### 컨트롤러
- `PageController.java`: `/profile` 요청을 `pages/profile` 뷰 대신 `/`(루트)로 리다이렉트 처리.

### UI 템플릿
- `templates/pages/profile.html`: 기존 독립 프로필 페이지(911줄) 삭제.
- `templates/pages/dashboard.html`: 프로필 섹션(About Me, 기술 스택, 자격증 등) 44줄 통합 추가.
- `templates/fragments/header.html`: 헤더 프로필 링크를 `/`로 수정.
- `templates/fragments/footer.html`: 푸터 링크 수정.
- `templates/pages/login.html`: 로그인 페이지 연결 링크 수정.

## 주요 구현 내용
로그인 없이도 개발자 프로필을 볼 수 있도록 UX를 개선했다. 기존 `/profile` URL로 접근하는 구버전 링크는 `PageController`에서 `/`로 리다이렉트하여 하위 호환성을 유지했다. `PublicEndpointConfig`의 페이지 엔드포인트 목록에 루트 경로를 추가하여 Spring Security 인증 필터를 우회할 수 있도록 처리했다.

## 주의사항
- `/profile` 경로는 삭제되지 않고 리다이렉트로 유지되므로, 외부에 공유된 프로필 링크가 있어도 정상 동작한다.
- 대시보드에 프로필 섹션이 통합되어 이후 프로필 콘텐츠 수정은 `dashboard.html`에서만 진행해야 한다.
