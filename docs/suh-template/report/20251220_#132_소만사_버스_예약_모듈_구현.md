## 📌 작업 개요

외부 프로젝트 `auto-bus-reservation`의 버스 예약 기능을 `suh-project-utility` 프로젝트에 통합. 기존 ENUM 기반 노선 관리를 DB 기반으로 변경하여 동적 관리 가능하도록 구현.

**주요 구현 내용**:
- `Suh-Domain-Somansa-Bus` 모듈 신규 생성
- 4개 Entity, 4개 Repository, 5개 Service 구현
- REST API Controller 및 Thymeleaf UI 페이지 구현
- 15개 버스 노선 초기 데이터 자동 생성

---

## 🎯 구현 목표

1. 버스 예약 사용자 동적 등록/관리
2. DB 기반 버스 노선 관리 (기존 ENUM 대체)
3. 사용자별 예약 스케줄 관리
4. 예약 성공/실패 이력 추적
5. 관리 UI 페이지 제공

### 2. Entity 구현

#### SomansaBusUser
- **파일**: `entity/SomansaBusUser.java`
- **필드**: somansaBusUserId(UUID), loginId, displayName, isActive, isVerified
- **설명**: 소만사 이메일 기반 사용자 정보 관리

#### SomansaBusRoute
- **파일**: `entity/SomansaBusRoute.java`
- **필드**: somansaBusRouteId(UUID), description, disptid, caralias, departureTime, station, busNumber, isActive
- **설명**: 버스 노선 정보 (출근/퇴근 구분, 출발시간, 정류장, 호차)

#### SomansaBusSchedule
- **파일**: `entity/SomansaBusSchedule.java`
- **필드**: somansaBusScheduleId(UUID), user(ManyToOne), route(ManyToOne), dayOfWeek, isActive
- **설명**: 사용자별 요일별 예약 스케줄

#### SomansaBusReservationHistory
- **파일**: `entity/SomansaBusReservationHistory.java`
- **필드**: somansaBusReservationHistoryId(UUID), user, route, schedule, reservationDate, isSuccess, errorMessage, httpStatus, responseBody
- **설명**: 예약 실행 결과 기록

### 4. Service 구현

#### SomansaBusApiService
- **파일**: `service/SomansaBusApiService.java`
- **기능**: 외부 버스 예약 시스템 API 연동
- **주요 메서드**: login(), createSession(), makeReservation()
- **특이사항**: OkHttp 기반 HTTP 클라이언트 사용

#### SomansaBusUserService
- **파일**: `service/SomansaBusUserService.java`
- **기능**: 사용자 등록/조회/삭제/활성화 토글
- **인증 방식**: API 로그인으로 소만사 직원 여부 확인 (suh-somansa-auth 대체)

#### SomansaBusRouteService
- **파일**: `service/SomansaBusRouteService.java`
- **기능**: 노선 조회 (전체, 타입별)

#### SomansaBusScheduleService
- **파일**: `service/SomansaBusScheduleService.java`
- **기능**: 스케줄 생성/조회/삭제/활성화 토글

#### SomansaBusReservationService
- **파일**: `service/SomansaBusReservationService.java`
- **기능**: 수동 예약 실행, 예약 이력 조회
- **자동 예약**: @Scheduled cron (매일 00:13) - 추후 구현 예정

## 🔧 주요 변경사항 상세

### suh-somansa-auth 의존성 이슈

**문제**: `suh-somansa-auth` 라이브러리가 JVM 21 필요, 현재 프로젝트는 JVM 17 사용

**해결**:
- Suh-Common/build.gradle에서 해당 의존성 주석 처리
- SomansaBusUserService에서 API 로그인으로 직원 인증 대체
- TODO 주석 추가: 프로젝트 업그레이드 후 활성화 예정

### isActive null 오류 수정

**문제**: SomansaBusDataInitializer에서 isActive 값 미설정으로 NOT NULL 제약조건 위반

**해결**: Builder에 `.isActive(true)` 추가
