### 📌 작업 개요
권한이 없는 사용자가 관리자 전용 페이지(`/notice-management`)에 접근할 때, `error/403.html` Thymeleaf 템플릿이 렌더링되지 않고 JSON 에러 응답이 표시되는 버그 수정. Spring Security의 403 처리 핸들러 추가 및 에러 페이지 라우팅 정상화.

### 🔍 문제 분석

**현상**: 비관리자가 `/notice-management` 접근 시 403 에러 페이지 대신 아래 JSON이 표시됨
```json
{
  "errorCode": "INTERNAL_SERVER_ERROR",
  "httpStatus": "INTERNAL_SERVER_ERROR",
  "message": "No static resource error/403."
}
```

**근본 원인 3가지**:

1. **PageController의 잘못된 리다이렉트**: `return "redirect:/error/403"` 사용 → `/error/403` URL에 매핑된 핸들러 없음 → `GlobalExceptionHandler`가 JSON으로 응답
2. **Spring Security exceptionHandling 미설정**: Security 레벨의 403(AccessDenied) 처리 핸들러가 없어서 기본 동작에 의존
3. **PublicEndpointConfig에 `/error/**` 경로 누락**: 미인증 사용자가 에러 페이지 자체에 접근 불가

**요청 흐름 (수정 전)**:
```
비관리자 → /notice-management → redirect:/error/403 → 매핑 없음 → 404 → GlobalExceptionHandler → JSON 에러
```

### ✅ 구현 내용

#### 1. PageController 리다이렉트 방식 수정
- **파일**: `Suh-Web/src/main/java/me/suhsaechan/web/controller/view/PageController.java`
- **변경 내용**: `redirect:/error/403` → 직접 Thymeleaf 템플릿 반환 + HTTP 403 상태코드 설정
- **이유**: 존재하지 않는 URL로 리다이렉트하는 대신, 403 상태코드와 함께 에러 템플릿을 직접 렌더링

```java
// 수정 전
return "redirect:/error/403";

// 수정 후
response.setStatus(HttpServletResponse.SC_FORBIDDEN);
return "error/403";
```

#### 2. Spring Security accessDeniedHandler 추가
- **파일**: `Suh-Web/src/main/java/me/suhsaechan/web/config/WebSecurityConfig.java`
- **변경 내용**: `exceptionHandling` 설정에 `accessDeniedHandler` 추가
- **이유**: Spring Security 레벨에서 발생하는 403(`.hasRole()` 등)도 `CustomErrorController`를 거쳐 `error/403.html` 템플릿을 렌더링하도록 처리

```java
http.exceptionHandling(exception -> exception
    .accessDeniedHandler((request, response, e) -> {
        response.sendError(HttpServletResponse.SC_FORBIDDEN);
    })
);
```

#### 3. 에러 경로 공개 접근 허용
- **파일**: `Suh-Web/src/main/java/me/suhsaechan/web/config/PublicEndpointConfig.java`
- **변경 내용**: `getPublicResourceEndpoints()`에 `/error/**` 경로 추가
- **이유**: 미인증 사용자도 에러 페이지를 볼 수 있도록 공개 경로에 포함

#### 4. 에러 페이지 Tailwind 하드코딩 값 수정
- **파일**: `Suh-Web/src/main/resources/templates/error/403.html`, `500.html`
- **변경 내용**: `min-h-[70vh]` → `min-h-screen` (Tailwind 표준 클래스로 변경)
- **이유**: 프로젝트 코딩 컨벤션(Tailwind 하드코딩 값 사용 금지) 준수. 404.html과 스타일 통일

#### 5. .gitignore에 .issue 폴더 추가
- **파일**: `.gitignore`
- **변경 내용**: `/.issue` 항목 추가
- **이유**: 이슈 작성용 로컬 파일이 Git에 추적되지 않도록 설정

### 🔧 주요 변경사항 상세

#### 에러 페이지 라우팅 흐름 (수정 후)
```
비관리자 → /notice-management → HTTP 403 + error/403.html 직접 렌더링
```

#### Spring Security 403 처리 흐름 (수정 후)
```
인가 실패 → accessDeniedHandler → sendError(403) → CustomErrorController → error/403.html
```

**특이사항**:
- `CustomErrorController`는 기존대로 `/error` 경로만 매핑. `sendError()`를 통해 서블릿 컨테이너가 `/error`로 내부 포워딩하므로 정상 동작
- `PageController`에서의 수동 권한 체크(`isSuperAdmin`)와 Spring Security의 URL 기반 권한 체크(`.hasRole()`)가 혼재되어 있으나, 양쪽 모두 403 에러 페이지가 정상 표시되도록 처리 완료

### 🧪 테스트 및 검증
- 비관리자 계정으로 `/notice-management` 접근 시 `error/403.html` 정상 렌더링 확인
- Spring Security에서 권한 거부된 경우에도 403 에러 페이지 표시 확인
- 미인증 사용자의 에러 페이지 접근 가능 여부 확인
- 기존 404, 500 에러 페이지 정상 동작 확인
