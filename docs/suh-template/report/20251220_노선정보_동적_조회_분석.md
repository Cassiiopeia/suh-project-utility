## 📌 분석 요약

현재 `BusRoute` ENUM에 하드코딩된 노선 정보를 DB에서 동적으로 조회하도록 전환하는 작업입니다. 예약 API 호출 시 필요한 `disptid`와 `caralias` 값을 DB에서 가져와 사용하도록 변경해야 합니다.

**감지된 프로젝트 타입**: Spring Boot 3.4.2 멀티 모듈 프로젝트  
**주요 기술 스택**: 
- Backend: Java 17, Spring Boot 3.4.2, Spring Data JPA
- Database: PostgreSQL
- Build: Gradle 8.12.1

**코드 스타일**: 
- 엔티티: `BasePostgresEntity` 상속, `@SuperBuilder`, `@Getter`, UUID 기반 ID
- 서비스: `@RequiredArgsConstructor` + 생성자 주입, `@Transactional(readOnly = true)` 기본
- Repository: Spring Data JPA 인터페이스 사용

## 📝 상세 구현 계획

### 1️⃣ DB 엔티티 설계

#### 1.1 BusRoute 엔티티 생성

**파일**: `Suh-Domain-BusReservation/src/main/java/me/suhsaechan/busreservation/entity/BusRoute.java`

**엔티티 구조**:
```java
@Entity
@Getter
@SuperBuilder
@NoArgsConstructor
@AllArgsConstructor
public class BusRoute extends BasePostgresEntity {
    
    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    @Column(nullable = false, updatable = false)
    private UUID busRouteId;
    
    /**
     * 노선 설명 (예: "06:55 당산역 1호 - 출근")
     */
    @Column(nullable = false)
    private String description;
    
    /**
     * 노선 ID (예약 API에 필요한 disptid)
     * 예: 46563, 46565 등
     */
    @Column(nullable = false, unique = true)
    private Integer disptid;
    
    /**
     * 셔틀 유형 (예약 API에 필요한 caralias)
     * "출근" 또는 "퇴근"
     */
    @Column(nullable = false, length = 10)
    private String caralias;
    
    /**
     * 출발 시간 (예: "06:55")
     */
    @Column(length = 10)
    private String departureTime;
    
    /**
     * 출발지 (예: "당산역", "군자", "서울역")
     */
    @Column(length = 50)
    private String departureLocation;
    
    /**
     * 노선 번호 (예: 1호, 2호, 3호)
     */
    private Integer routeNumber;
    
    /**
     * 노선 활성화 여부
     */
    @Builder.Default
    private Boolean isActive = true;
    
    /**
     * 노선 정보 업데이트
     */
    public void update(String description, Integer disptid, String caralias, 
                      String departureTime, String departureLocation, 
                      Integer routeNumber, Boolean isActive) {
        this.description = description;
        this.disptid = disptid;
        this.caralias = caralias;
        this.departureTime = departureTime;
        this.departureLocation = departureLocation;
        this.routeNumber = routeNumber;
        this.isActive = isActive;
    }
}
```

**설계 고려사항**:
- `disptid`는 UNIQUE 제약조건 (중복 방지)
- `caralias`는 "출근" 또는 "퇴근"만 허용 (추후 Validation 추가 가능)
- `isActive`로 노선 활성화/비활성화 관리
- `BasePostgresEntity` 상속으로 생성일/수정일 자동 관리

@Service
@RequiredArgsConstructor
@Transactional(readOnly = true)
public class BusRouteService {
    
    private final BusRouteRepository busRouteRepository;
    
    /**
     * 활성화된 모든 노선 조회

     */
    public List<BusRoute> getRoutesByCaralias(String caralias) {
        return busRouteRepository.findByIsActiveTrueAndCaralias(caralias);
    }

### 3️⃣ 초기 데이터 마이그레이션

#### 3.1 데이터 초기화 서비스

**파일**: `Suh-Domain-BusReservation/src/main/java/me/suhsaechan/busreservation/service/BusRouteInitService.java`

**기능**:
- 기존 ENUM 데이터를 DB에 초기 데이터로 삽입
- 애플리케이션 시작 시 또는 별도 초기화 API로 실행

**구현 방법 1: @PostConstruct 사용**
```java
@Service
@RequiredArgsConstructor
@Slf4j
public class BusRouteInitService {
    
    private final BusRouteRepository busRouteRepository;
    
    @PostConstruct
    public void initBusRoutes() {
        // 이미 데이터가 있으면 스킵
        if (busRouteRepository.count() > 0) {
            log.info("버스 노선 데이터가 이미 존재합니다. 초기화를 건너뜁니다.");
            return;
        }
        
        // ENUM 데이터를 DB에 삽입
        List<BusRoute> routes = Arrays.asList(
            BusRoute.builder()
                .description("06:55 당산역 1호 - 출근")
                .disptid(46563)
                .caralias("출근")
                .departureTime("06:55")
                .departureLocation("당산역")
                .routeNumber(1)
                .isActive(true)
                .build(),
            // ... 나머지 12개 노선
        );
        
        busRouteRepository.saveAll(routes);
        log.info("버스 노선 초기 데이터 {}개 삽입 완료", routes.size());
    }
}
```

**구현 방법 2: 별도 초기화 API 제공**
```java
@RestController
@RequestMapping("/api/bus-reservation/admin")
@RequiredArgsConstructor
public class BusRouteAdminController {
    
    private final BusRouteInitService initService;
    
    @PostMapping("/init-routes")
    public ResponseEntity<String> initRoutes() {
        initService.initBusRoutes();
        return ResponseEntity.ok("버스 노선 초기화 완료");
    }
}
```

**권장**: 방법 1 (@PostConstruct) - 애플리케이션 시작 시 자동 실행

## ⚠️ 주의사항

### 1. ENUM 제거 시점
- DB 마이그레이션 완료 후 ENUM 제거
- 모든 코드에서 ENUM 참조 제거 확인 필요
- 컴파일 에러 발생 시점에 수정

### 2. 데이터 일관성
- 초기 데이터 마이그레이션 시 ENUM 데이터와 일치하도록 주의
- `disptid` 값이 정확히 일치해야 예약 API 정상 동작

### 3. Null 처리
- DB에서 노선 조회 실패 시 적절한 예외 처리
- `Optional.orElseThrow()` 사용 권장

### 4. 성능 고려
- 노선 정보는 자주 변경되지 않으므로 캐싱 고려 가능
- `@Cacheable` 어노테이션 활용 검토

### 5. 하위 호환성
- 기존 API가 `disptid`를 직접 받는 경우 대비
- `disptid`로 노선 조회하는 메서드 제공

### 6. 테스트 코드 수정
- Mock 객체를 ENUM에서 엔티티로 변경
- Repository Mock 설정 필요
