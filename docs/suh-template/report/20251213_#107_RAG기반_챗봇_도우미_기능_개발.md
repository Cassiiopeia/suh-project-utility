### 🔧 주요 변경사항 상세

#### Qdrant 벡터 데이터베이스 연동

- **클라이언트**: `io.qdrant:client:1.7.2` (gRPC 기반)
- **컬렉션**: `suh_chatbot_documents` (벡터 크기: 768)
- **Distance**: Cosine 유사도
- **환경별 설정**:
  - dev: `localhost:6334`
  - prod: `suh-project.synology.me:16334`

#### MockEmbeddingService

LLM 연동 전까지 임시로 768차원 랜덤 벡터 생성. 향후 Ollama embedding-gemma로 교체 예정.

#### ChatbotService 핵심 로직

1. 세션 생성/조회 (토큰 기반)
2. 사용자 메시지 저장
3. 임베딩 생성 → Qdrant 검색
4. 컨텍스트 기반 응답 생성 (현재 Mock)
5. AI 응답 저장 및 반환

**특이사항**:
- 검색 결과가 없으면 "관련 정보를 찾지 못했습니다" 메시지 반환
- 피드백은 ChatMessage 엔티티에 isHelpful 필드로 저장

### 📌 참고사항

#### 향후 작업 예정

1. **MockEmbeddingService → OllamaEmbeddingService**: Ollama embedding-gemma 모델 연동
2. **MockLlmService → 실제 LLM 연동**: 응답 생성 로직 구현
3. **관리자 문서 등록 페이지**: ChatDocument CRUD UI 개발
4. **챗봇 아바타 이미지**: 사용자 제공 이미지로 교체 예정

#### 기술 스택

- **Backend**: Spring Boot 3.4.2, Spring Data JPA
- **Vector DB**: Qdrant 1.16.2 (Docker)
- **Frontend**: Thymeleaf, jQuery, Tailwind CSS
- **통신**: FormData 기반 AJAX (기존 프로젝트 패턴 준수)

#### UI/UX 개선 이력

1. **초기 버전**: 헤더 포함, FAB에 X 아이콘 겹침 문제
2. **개선 버전**: 헤더 제거, FAB 단순화, 이름 "SuhNi(서니)"로 변경
3. **최종 버전**: 초기화 버튼 추가, 웰컴 화면 복귀 기능 구현

#### 데이터베이스 스키마

**ChatDocument** (문서 메타데이터)
- chatDocumentId (UUID)
- title, category, content (TEXT)
- isActive, isProcessed (Boolean)
- chunkCount, orderIndex

**ChatDocumentChunk** (문서 청크)
- chatDocumentChunkId (UUID)
- chatDocument (FK)
- pointId (UUID) - Qdrant 포인트 ID
- chunkIndex, content (TEXT)
- tokenCount

**ChatSession** (채팅 세션)
- chatSessionId (UUID)
- sessionToken (String)
- userIp, userAgent
- lastActivityAt, isActive
- messageCount, title

**ChatMessage** (채팅 메시지)
- chatMessageId (UUID)
- chatSession (FK)
- role (USER/ASSISTANT/SYSTEM)
- content (TEXT), messageIndex
- referencedDocumentIds (JSON)
- inputTokens, outputTokens
- responseTimeMs, isHelpful
