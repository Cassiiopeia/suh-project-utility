## 3. 우선순위별 발견 사항

### 3.1 P0 — Critical (CLAUDE.md 절대 규칙 위반)

#### [P0-1] `@OneToMany` + `cascade = CascadeType.ALL` 위반 (3건)

CLAUDE.md "절대 금지" 명시 규칙 위반. 모두 Study 도메인.

| 위치 | 코드 | 위반 |
|------|------|------|
| `Suh-Domain-Study/.../StudyPost.java:80` | `@OneToMany(mappedBy="post", cascade=CascadeType.ALL, orphanRemoval=true) attachments` | @OneToMany + cascade |
| `Suh-Domain-Study/.../StudyCategory.java:67` | `@OneToMany(...) children` (자기참조) | @OneToMany + cascade |
| `Suh-Domain-Study/.../StudyCategory.java:74` | `@OneToMany(...) posts` | @OneToMany + cascade |

**연관 정보**: 이슈 #176에서 Study 모듈은 이미 제거됨 (`settings.gradle` include 제거, Application/Web 의존성 제거). 단 디렉터리/소스 파일이 잔존 → 빌드에선 빠지지만 코드베이스에 노이즈.

#### [P0-2] Response DTO 금지 필드 사용 (2건)

CLAUDE.md "API 응답 규칙 절대 준수" 위반.

| 위치 | 필드 |
|------|------|
| `Suh-Web/.../dto/SejongAuthResponse.java:16` | `isSuccess` |
| `Suh-Web/.../dto/SejongAuthResponse.java:38` | `errorMessage` |

응답 패턴: 성공 시 데이터만 / 실패 시 `CustomException` + `GlobalExceptionHandler` 위임.

#### [P0-3] Study 모듈 디렉터리 cleanup 미완

이슈 #176 이미 처리됐으나 `Suh-Domain-Study/` 폴더 + 소스 잔존. 빌드 영향 없음 (settings.gradle 미등록). 단:
- P0-1 위반 코드가 그대로 남아 정적 분석에서 false positive 발생
- 신규 개발자 혼란 가능

### 3.2 P1 — Major (아키텍처 결함)

#### [P1-1] 도메인 모듈 → Web 모듈 역방향 테스트 의존

전 도메인 모듈이 `testImplementation project(':Suh-Web')` 선언. `@SpringBootTest(classes = SuhProjectUtilityApplication.class)` 호출 목적.

| 항목 | 영향 |
|------|------|
| 레이어 침범 | Common ← Domain ← Application ← Web 흐름에 역방향 (test scope) |
| 빌드 시간 | 단일 도메인 테스트 위해 Web 전체 빌드 필요 |
| 격리도 | 도메인 단위 독립 검증 불가 |

#### [P1-2] Suh-Application 레이어 효용 의문

| 사실 |
|------|
| 멤버 클래스: `DashboardService.java` 1개 |
| 끌어오는 도메인: 7개 (Docker, Github, Chatbot, GrassPlanter, AiServer, Statistics, Translate) |
| Web도 동일 도메인 7개를 직접 의존 → 우회 경로 |
| Notice, Somansa-Bus, Task는 Application 미포함 (포함 기준 일관성 없음) |

#### [P1-3] Suh-Web 비대화

- Config 클래스 16개
- Controller 19개 (API 14 + View 2 + Error 1)
- Thymeleaf 템플릿 23개
- 모든 HTTP 진입점이 Web 단일 모듈에 집중

도메인 모듈이 자기 Controller/View 미보유 → "도메인 모듈" 명명과 실제 책임 불일치. 사실상 service-layer 모듈에 가까움.

#### [P1-4] ServerOptionKey 활용 저조

| 카테고리 | 카운트 | 비율 |
|----------|--------|------|
| `ServerOptionKey` enum | 5개 | 12% |
| `@Value` 주입 | 36건 | 88% |

`@Value` 중 운영 중 변경 가능 후보(재배포 없이 바꾸고 싶은 값):
- `ai-server.base-url`
- `qdrant.collection-name`, `qdrant.vector-size`
- `vertex.ai.model`, `vertex.ai.dimension`
- 챗봇 청크 관련 파라미터(이미 ServerOptionKey 사용 — 좋은 사례)

#### [P1-5] Optional 패턴 광범위 위반

- `== null` / `!= null` 비교 사용 파일 수: 54개
- 누적 발생: 200건 이상
- 최악 단일 파일: `AiServerService.java` 30+ 건

CLAUDE.md: "null 체크 금지, `orElseThrow()/orElse()/orElseGet()` 사용".

## 7. 주의사항

- **Study 디렉터리 삭제**는 빌드 영향 없음 확인됨(이슈 #176에서 이미 settings.gradle 제외). 단순 정리.
- **Application 레이어 제거** 진행 시 7개 도메인 import 위치 전부 추적 필요 — Suh-Web 의존성 추가만 하면 회귀 없음(이미 직접 의존 중).
- **ArchUnit 도입** 시 기존 위반 즉시 빌드 실패 → 점진 도입(warning 모드 시작) 필수.
- **`ddl-auto: validate` 전환**은 prod DB 스키마와 Entity 100% 일치해야 함. 사전 비교 후 진행.
- 본 분석 범위 **테스트 코드 품질/커버리지 미조사**. 별도 `/cassiiopeia:review` 또는 `/cassiiopeia:test` 호출 권장.
