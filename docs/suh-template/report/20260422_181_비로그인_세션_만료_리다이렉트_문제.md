# [버그][인증] 비로그인 상태에서 세션 만료 시 로그인 페이지로 리다이렉트되는 문제

## 개요
비로그인 상태에서 세션이 만료(invalidSession)될 경우 로그인 페이지로 리다이렉트되던 문제를 수정했다. `invalidSessionUrl`을 대시보드(`/`)로 변경하여 비로그인 사용자가 불필요하게 로그인 페이지로 이동하지 않도록 개선했다. 중복 로그인으로 인한 세션 만료(`expiredUrl`)는 기존대로 로그인 페이지 리다이렉트를 유지한다.

## 변경 사항

### 보안 설정
- `Suh-Web/src/main/java/me/suhsaechan/web/config/WebSecurityConfig.java`: `sessionManagement` 설정에서 `invalidSessionUrl`을 `/login?sessionExpired=true`에서 `/`(대시보드)로 변경. `expiredUrl`은 중복 로그인 세션 만료 케이스이므로 `/login?sessionExpired=true` 유지.

## 주요 구현 내용
Spring Security의 세션 관리에서 `invalidSession`과 `expiredSession`은 서로 다른 케이스다. `invalidSessionUrl`은 유효하지 않은 세션 ID를 가진 요청(비로그인 포함)에 적용되고, `expiredUrl`은 동일 계정이 다른 기기에서 로그인하여 기존 세션이 만료된 경우에 적용된다. 이번 수정으로 비로그인 사용자가 세션 쿠키 없이 접근할 때 대시보드로 이동하며, 중복 로그인 케이스는 로그인 페이지로 안내하는 정책을 분리했다.

## 주의사항
- `invalidSessionUrl` 변경으로 비로그인 접근 시 항상 대시보드(`/`)로 이동하므로, 대시보드는 반드시 비인증 접근이 허용되어야 한다(이슈 #179에서 이미 처리됨).
