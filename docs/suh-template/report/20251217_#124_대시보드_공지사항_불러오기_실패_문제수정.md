### 📌 작업 개요
대시보드 진입 시 `/api/notice/get/active` API 호출에서 500 에러 발생하는 문제 수정.
Jackson이 `LocalDateTime` 타입을 직렬화하지 못해 발생한 문제로, `JacksonConfig`를 통해 전역 `ObjectMapper` 설정으로 해결.

### 🔍 문제 분석

**에러 메시지:**
```
org.springframework.http.converter.HttpMessageConversionException:
Type definition error: [simple type, class java.time.LocalDateTime]

Caused by: com.fasterxml.jackson.databind.exc.InvalidDefinitionException:
Java 8 date/time type `java.time.LocalDateTime` not supported by default
(through reference chain: me.suhsaechan.notice.dto.NoticeResponse["notices"]
->java.util.ArrayList[0]->me.suhsaechan.notice.entity.SuhProjectUtilityNotice["createdDate"])
```

**원인 분석:**
- `NoticeResponse`에서 `List<SuhProjectUtilityNotice>` Entity를 직접 반환
- `SuhProjectUtilityNotice`는 `BasePostgresEntity`를 상속하며 `LocalDateTime` 타입의 `createdDate`, `updatedDate` 필드 보유
- Entity 자체에도 `startDate`, `endDate` 필드가 `LocalDateTime` 타입
- Jackson이 기본적으로 `LocalDateTime`을 JSON으로 직렬화하는 방법을 모르기 때문에 에러 발생
- Spring Boot는 `jackson-datatype-jsr310` 모듈을 자동 등록하지만, 직렬화 포맷 설정이 필요함

**재현 방법:**
1. 로그인 후 대시보드 진입
2. `/api/notice/get/active` API 호출 시 500 에러 발생

### ✅ 구현 내용

#### JacksonConfig - 전역 ObjectMapper 설정 추가
- **파일**: `Suh-Web/src/main/java/me/suhsaechan/web/config/JacksonConfig.java` (신규 생성)
- **변경 내용**: `JavaTimeModule`을 사용하여 `LocalDateTime`, `LocalDate`, `LocalTime` 직렬화 포맷 전역 설정
- **이유**: 모든 Entity의 `LocalDateTime` 필드에 일괄 적용되며, Entity에 개별 어노테이션 추가 불필요

**구현 코드:**
```java
@Configuration
public class JacksonConfig {

    private static final String DATE_TIME_FORMAT = "yyyy-MM-dd'T'HH:mm:ss";
    private static final String DATE_FORMAT = "yyyy-MM-dd";
    private static final String TIME_FORMAT = "HH:mm:ss";

    @Bean
    @Primary
    public ObjectMapper objectMapper() {
        ObjectMapper objectMapper = new ObjectMapper();

        JavaTimeModule javaTimeModule = new JavaTimeModule();

        // LocalDateTime 직렬화 포맷
        javaTimeModule.addSerializer(
            java.time.LocalDateTime.class,
            new LocalDateTimeSerializer(DateTimeFormatter.ofPattern(DATE_TIME_FORMAT))
        );

        // LocalDate 직렬화 포맷
        javaTimeModule.addSerializer(
            java.time.LocalDate.class,
            new LocalDateSerializer(DateTimeFormatter.ofPattern(DATE_FORMAT))
        );

        // LocalTime 직렬화 포맷
        javaTimeModule.addSerializer(
            java.time.LocalTime.class,
            new LocalTimeSerializer(DateTimeFormatter.ofPattern(TIME_FORMAT))
        );

        objectMapper.registerModule(javaTimeModule);

        // 타임스탬프 형식 비활성화 (ISO-8601 문자열 형식 사용)
        objectMapper.disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);

        return objectMapper;
    }
}
```

### 🔧 주요 변경사항 상세

#### 전역 설정 방식 선택 이유
- **Entity 개별 설정 불필요**: 모든 Entity의 `LocalDateTime` 필드에 자동 적용
- **일관성 보장**: 프로젝트 전체에서 동일한 날짜/시간 포맷 사용
- **유지보수 용이**: 포맷 변경 시 한 곳만 수정하면 됨
- **Spring Boot 자동 감지**: `@Configuration` + `@Bean` + `@Primary`로 Spring이 자동으로 사용

**특이사항**:
- `@Primary` 어노테이션으로 기본 `ObjectMapper`로 등록되어 모든 JSON 직렬화에 적용
- `JavaTimeModule`은 Spring Boot에 포함되어 있어 별도 의존성 추가 불필요
- `WRITE_DATES_AS_TIMESTAMPS` 비활성화로 타임스탬프 대신 문자열 형식 사용
- `LocalDateTime`: `"yyyy-MM-dd'T'HH:mm:ss"` 형식 (예: `"2025-12-17T01:10:30"`)
- `LocalDate`: `"yyyy-MM-dd"` 형식 (예: `"2025-12-17"`)
- `LocalTime`: `"HH:mm:ss"` 형식 (예: `"01:10:30"`)

#### 적용 범위
- 모든 REST API 응답의 `LocalDateTime` 필드
- `BasePostgresEntity`의 `createdDate`, `updatedDate`
- 모든 Entity의 `LocalDateTime` 타입 필드 (`startDate`, `endDate` 등)
- DTO의 `LocalDateTime` 필드

### 📦 의존성 변경
- 없음 (Spring Boot에 포함된 `jackson-datatype-jsr310` 사용)
- `jackson-databind`는 이미 `Suh-Common/build.gradle`에 포함되어 있음

### 🧪 테스트 및 검증
**수정 후 확인 사항:**
1. `./gradlew compileJava` 빌드 성공 확인
2. 대시보드 진입 시 공지사항 정상 로드 확인
3. `/api/notice/get/active` API 호출 시 500 에러 해결 확인
4. 응답 JSON에서 `createdDate`, `updatedDate`, `startDate`, `endDate` 필드가 `"yyyy-MM-dd'T'HH:mm:ss"` 형식으로 직렬화되는지 확인
5. 다른 API 엔드포인트에서도 `LocalDateTime` 필드가 정상적으로 직렬화되는지 확인

### 📌 참고사항
- 전역 설정이므로 모든 Entity의 `LocalDateTime` 필드에 자동 적용됨
- 향후 새로운 Entity에 `LocalDateTime` 필드 추가 시 별도 설정 불필요
- 포맷 변경이 필요한 경우 `JacksonConfig.java`의 상수만 수정하면 됨
- `@JsonFormat` 어노테이션을 Entity에 추가할 필요 없음 (전역 설정이 우선 적용)
