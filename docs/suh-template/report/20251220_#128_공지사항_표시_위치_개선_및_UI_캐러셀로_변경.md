### 📌 작업 개요
공지사항이 너무 많은 화면 공간을 차지하는 문제 해결. 헤더와 공지사항을 완전 분리하고, 슬라이드 캐러셀 형태로 변경하여 공간 효율성 개선

**보고서 파일**: `.report/20251220_#128_공지사항_표시_위치_개선_및_UI_캐러셀로_변경.md`

### 🎯 구현 목표
- 공지사항 표시 영역 축소로 화면 공간 효율성 향상
- 헤더와 공지사항 섹션 완전 분리
- 슬라이드 캐러셀 형태로 여러 공지사항 표시
- 중요 공지사항 우선 노출 및 최신 공지사항 자동 슬라이드
- daisyUI 캐러셀 참고하되 커스텀 애니메이션 구현

### ✅ 구현 내용

#### 헤더와 공지사항 섹션 분리
- **파일**: `Suh-Web/src/main/resources/templates/pages/dashboard.html`
- **변경 내용**: `header-section`과 `notice-section`을 독립적인 섹션으로 분리
- **이유**: 헤더와 공지사항의 역할을 명확히 구분하고, 공지사항 섹션만 독립적으로 관리 가능하도록 구조 개선

#### 공지사항 슬라이드 카드 레이아웃 구현
- **파일**: `Suh-Web/src/main/resources/templates/pages/dashboard.html`, `Suh-Web/src/main/resources/static/css/common.css`
- **변경 내용**: 공지사항을 높이 4rem 고정 슬라이드 카드 형태로 변경
- **이유**: 기존 카드 형태에서 슬라이드 형태로 변경하여 공간을 절약하면서도 여러 공지사항을 순환 표시 가능

#### 자동 슬라이드 애니메이션 구현
- **파일**: `Suh-Web/src/main/resources/templates/pages/dashboard.html`
- **변경 내용**: 5초마다 자동으로 다음 공지사항으로 슬라이드되는 기능 추가
- **이유**: 사용자가 여러 공지사항을 자동으로 확인할 수 있도록 구현

#### 중요 공지사항 우선 정렬 로직
- **파일**: `Suh-Web/src/main/resources/templates/pages/dashboard.html`
- **변경 내용**: 중요 공지사항을 우선 표시하고, 그 다음 최신순으로 정렬
- **이유**: 중요한 공지사항이 먼저 노출되도록 구현

### 🔧 주요 변경사항 상세

#### dashboard.html - 공지사항 섹션 구조 변경
기존 카드 형태에서 슬라이드 카드 컨테이너로 변경. 높이 4rem으로 고정하여 공간 효율성 확보

**주요 변경점**:
- `notice-section`을 독립 섹션으로 분리
- `notice-cards-container` 추가 (높이 4rem 고정)
- 공지사항 헤더(레이블 + 전체보기 링크) 고정 표시
- 슬라이드 카드들이 절대 위치로 겹쳐서 표시되도록 구조 변경

#### common.css - 슬라이드 카드 스타일 및 애니메이션
커스텀 슬라이드 애니메이션 구현. daisyUI 캐러셀을 참고하되 자체 구현

**주요 스타일**:
- `.notice-card`: 절대 위치, 초기 상태는 `translateY(100%)`로 아래에 숨김
- `.notice-card.active`: `translateY(0)`으로 화면에 표시
- `.notice-card.exit`: `translateY(-100%)`로 위로 사라지는 애니메이션
- `transition: transform 0.6s cubic-bezier(0.4, 0, 0.2, 1)`로 부드러운 애니메이션

**특이사항**:
- 다크모드 지원 (배경색, 텍스트 색상 자동 변경)
- 모바일 반응형 (높이 3rem, 메타 정보 숨김)
- 호버 시 색상 변경으로 인터랙션 피드백 제공

#### JavaScript - 슬라이드 로직 구현
자동 슬라이드 및 수동 제어 기능 구현

**주요 함수**:
- `initNoticeCards()`: 공지사항 데이터를 기반으로 카드 생성
- `startNoticeSlide()`: 5초마다 자동 슬라이드 시작
- `slideToNext()`: 다음 카드로 전환 (현재 카드는 위로, 다음 카드는 아래에서 올라옴)
- `pauseNoticeSlide()` / `resumeNoticeSlide()`: 호버 시 일시정지 및 재개

**특이사항**:
- 2개 이상의 공지사항이 있을 때만 자동 슬라이드 작동
- 호버 시 자동 슬라이드 일시정지로 사용자 편의성 향상
- 클릭 시 상세 모달 열기 및 슬라이드 일시정지

#### 정렬 로직
중요 공지사항 우선, 그 다음 최신순 정렬

```javascript
noticeData.sort(function(a, b) {
  if (a.isImportant && !b.isImportant) return -1;
  if (!a.isImportant && b.isImportant) return 1;
  return new Date(b.createdDate) - new Date(a.createdDate);
});
```

### 🧪 테스트 및 검증
- 공지사항이 1개일 때: 자동 슬라이드 비활성화 확인
- 공지사항이 2개 이상일 때: 5초마다 자동 슬라이드 확인
- 중요 공지사항이 있을 때: 중요 공지사항이 먼저 표시되는지 확인
- 호버 시: 슬라이드 일시정지 및 재개 동작 확인
- 클릭 시: 상세 모달 정상 작동 확인
- 모바일: 반응형 레이아웃 정상 작동 확인
- 다크모드: 색상 테마 정상 적용 확인

### 📌 참고사항
- 공지사항 높이: 데스크탑 4rem, 모바일 3rem
- 슬라이드 간격: 5초
- 애니메이션 지속시간: 0.6초 (cubic-bezier 이징)
