### 📌 작업 개요
챗봇에서 페이지 이동 시 대화 내용이 사라지는 문제 해결. localStorage를 활용하여 대화 히스토리를 저장하고 페이지 로드 시 복원하는 기능 구현.

**이슈**: [#136](https://github.com/Cassiiopeia/suh-project-utility/issues/136)

### 🔍 문제 분석
Spring Boot + Thymeleaf 기반 MPA(Multi-Page Application) 환경에서 페이지 이동 시 전체 HTML이 새로 로드되어 DOM이 초기화됨. 챗봇 위젯도 매번 새로 렌더링되어 이전 대화 내용이 사라지는 현상 발생.

**기술적 한계**:
- MPA 환경에서는 페이지 이동 시 DOM 유지 불가능
- SPA와 달리 JavaScript 상태가 페이지 간 공유되지 않음
- **해결 방향**: "유지" 대신 "복원" 방식 채택 (localStorage 활용)

### ✅ 구현 내용

#### 대화 히스토리 저장 함수 추가
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**: `saveHistory()` 함수 추가
- **동작**: DOM에서 모든 채팅 메시지를 읽어 localStorage에 JSON 형태로 저장

#### 대화 히스토리 복원 함수 추가
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**: `restoreHistory()` 함수 추가
- **동작**: 페이지 로드 시 localStorage에서 저장된 메시지를 읽어 채팅창에 복원

#### 히스토리 전용 메시지 추가 함수
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**: `appendMessageFromHistory()` 함수 추가
- **이유**: 복원 시 `saveHistory()` 재호출 방지를 위해 별도 함수 분리

#### 초기화 시 히스토리 복원
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**: `init()` 함수에 `restoreHistory()` 호출 추가
- **동작**: 페이지 로드 시 자동으로 이전 대화 내용 복원

#### 메시지 추가 시 자동 저장
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**: `appendMessage()`, `finalizeStreamingMessage()` 함수에 `saveHistory()` 호출 추가
- **동작**: 사용자 메시지 전송, AI 응답 완료 시 자동으로 히스토리 저장

#### 초기화 버튼 클릭 시 히스토리 삭제
- **파일**: `Suh-Web/src/main/resources/static/js/chatbot.js`
- **변경 내용**: `resetChat()` 함수에 `localStorage.removeItem('chatbot_history')` 추가
- **동작**: "새 대화" 버튼 클릭 시 저장된 히스토리도 함께 삭제

### 🔧 주요 변경사항 상세

#### localStorage 데이터 구조
```javascript
{
  sessionToken: "abc123",
  messages: [
    { role: "user", content: "안녕하세요", time: "14:30" },
    { role: "assistant", content: "안녕하세요! 무엇을 도와드릴까요?", time: "14:30" }
  ]
}
```

#### 동작 흐름
```
[페이지 로드]
     ↓
init() → restoreHistory() → localStorage에서 메시지 복원
     ↓
[사용자 대화]
     ↓
appendMessage() → saveHistory() (사용자 메시지 저장)
     ↓
finalizeStreamingMessage() → saveHistory() (AI 응답 저장)
     ↓
[페이지 이동]
     ↓
[새 페이지 로드]
     ↓
init() → restoreHistory() → 이전 대화 보임 ✅
     ↓
[reset 버튼 클릭]
     ↓
resetChat() → localStorage 삭제 → 빈 상태
```

**특이사항**:
- 마크다운 렌더링된 HTML이 아닌 텍스트만 저장하여 용량 최소화
- 복원 시 다시 마크다운 렌더링 적용
- localStorage 용량 제한(5MB) 내에서 충분히 동작

### 🧪 테스트 및 검증
1. 챗봇에서 여러 대화 진행
2. 다른 페이지로 이동 후 복귀
3. 챗봇 열기 → **이전 대화가 그대로 표시되면 성공**
4. reset 버튼 클릭 → **대화 삭제 및 웰컴 메시지 표시 확인**

### 📌 참고사항
- 브라우저 localStorage 삭제 시 대화 내용도 함께 삭제됨
- 다른 브라우저/기기에서는 대화 내용 공유 안 됨 (클라이언트 저장 방식 한계)
- 향후 서버 기반 히스토리 저장으로 확장 가능
