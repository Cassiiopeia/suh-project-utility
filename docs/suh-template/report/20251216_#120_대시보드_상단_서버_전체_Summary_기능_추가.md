# 대시보드 상단 서버 전체 Summary 기능 추가

## 📌 작업 개요

대시보드 상단에 서버 전체 통계 Summary 카드 추가. 방문자 수, 챗봇 대화 수, 토큰 사용량, 콘텐츠 조회수를 한눈에 파악할 수 있는 통계 대시보드 구현.

**이슈**: [#120](https://github.com/Cassiiopeia/suh-project-utility/issues/120)

---

## 🎯 구현 목표

- 대시보드 상단에 서버 전체 오버뷰 Summary 표시
- 방문자 조회수, 챗봇 사용률, 토큰 사용량 통계 수집 및 표시
- DaisyUI stats 컴포넌트 활용하여 shadcn 스타일의 모던한 UI 구현
- 향후 실시간 접속자, 시간대별 차트 등 확장 가능한 구조 설계

### 2. 챗봇 토큰 사용량 기록

**파일**: `Suh-Domain-Chatbot/.../ChatbotService.java`

기존 `ChatMessage`에 존재하던 `inputTokens`, `outputTokens` 필드에 실제 값 저장 로직 추가

**주요 변경**:
- `estimateTokenCount()` 메서드 추가 - 한글 기준 약 3자 = 1토큰으로 추정
- `chat()` 메서드에서 응답 저장 시 토큰 및 `responseTimeMs` 기록
- `chatStream()` 메서드에서도 동일하게 토큰 기록
- 프롬프트 전체 길이를 입력 토큰으로 계산
- AI 응답 내용 길이를 출력 토큰으로 계산

**토큰 추정 로직**:
```java
private static final int TOKEN_ESTIMATE_DIVISOR = 3;  // 한글 기준 약 3자 = 1토큰

private int estimateTokenCount(String text) {
    if (text == null || text.isEmpty()) return 0;
    return text.length() / TOKEN_ESTIMATE_DIVISOR;
}
```

**특이사항**:
- 정확한 토큰 계산이 아닌 추정값 사용 (한글 3자 = 1토큰)
- 실제 LLM 모델별 토큰 계산 방식과 차이 있을 수 있음
- 향후 tiktoken 라이브러리 연동으로 정확도 향상 가능

### 4. DashboardService (Application 모듈)

**파일**: `Suh-Application/.../service/DashboardService.java`

여러 도메인 모듈의 통계 데이터를 집계하여 Summary 제공

**집계 항목**:
- 방문자 통계 (총/오늘 고유 방문자, 페이지뷰)
- 챗봇 통계 (총/오늘 세션, 메시지 수)
- 토큰 사용량 (총/오늘 입력/출력 토큰)
- 콘텐츠 조회수 (공지사항 + 스터디 총 조회수)

**캐싱 전략**:
- `@Cacheable` 어노테이션으로 5분간 캐싱
- `refreshDashboardSummary()` 메서드로 캐시 무효화 후 새 데이터 조회
- Redis 기반 캐싱 (RedisConfig 설정 필요)

### 6. 페이지 방문 기록 Interceptor

**파일**: `Suh-Web/.../config/PageVisitInterceptor.java`

**기능**:
- 주요 페이지 방문 시 비동기로 기록 (`@Async`)
- 봇 탐지 (User-Agent 패턴 매칭: bot, crawl, spider, googlebot, bingbot 등)
- API, 정적 리소스 경로 제외
- 화이트리스트 기반 기록 대상 페이지 관리
- 클라이언트 해시 생성 (세션 기반 또는 IP+User-Agent 해시)

**클라이언트 해시 생성 로직**:
- 세션에 `clientHash`가 있으면 사용
- 없으면 IP + User-Agent를 SHA-256 해시하여 `anon_` 접두사와 함께 저장
- 동일 디바이스/브라우저 식별을 위한 고유 해시

**기록 대상 페이지**:
- `/`, `/dashboard`, `/login`, `/profile`, `/study`, `/notice`, `/docker`, `/github`, `/grass`, `/translate`, `/chatbot`, `/ai-server`

**WebConfig.java**에 인터셉터 등록:
```java
registry.addInterceptor(pageVisitInterceptor)
        .addPathPatterns("/**")
        .excludePathPatterns("/api/**", "/static/**", "/css/**", "/js/**", "/images/**");
```

### 7. Dashboard UI

**파일**: `Suh-Web/.../templates/pages/dashboard.html`

DaisyUI `stats` 컴포넌트 활용한 Summary 카드 추가

**UI 구성**:
```
┌─────────────────────────────────────────────────────────────────────┐
│         📊 서버 통계 Summary (그라데이션 배경)                         │
├─────────────────┬─────────────────┬─────────────────┬───────────────┤
│  👥 총 방문자    │  💬 챗봇 대화    │  ⚡ 토큰 사용량  │  👁 콘텐츠 조회 │
│    1.2K         │    567          │   12.5K         │    89         │
│   오늘: +23     │   오늘: +12     │   오늘: +2.3K   │  공지+스터디   │
└─────────────────┴─────────────────┴─────────────────┴───────────────┘
```

**특징**:
- 스켈레톤 로딩 UI 제공 (로딩 중 카드 형태의 스켈레톤 표시)
- 숫자 포맷 함수 (`formatNumber`: 1000 → `1K`, 1000000 → `1M`)
- 토큰 포맷 함수 (`formatTokens`: 동일한 포맷 적용)
- 새로고침 버튼 (회전 애니메이션, 캐시 무효화 후 재조회)
- 그라데이션 배경 (`from-blue-600 via-indigo-600 to-purple-600`)
- 반응형 레이아웃 (`grid-cols-1 sm:grid-cols-2 lg:grid-cols-4`)
- 호버 효과 (`hover:shadow-md transition-shadow`)

**JavaScript 함수**:
- `loadDashboardSummary()` - 페이지 로드 시 Summary 조회
- `refreshDashboardSummary()` - 새로고침 버튼 클릭 시 캐시 무효화 후 재조회
- `formatNumber(num)` - 숫자 포맷팅 (K, M 단위)
- `formatTokens(tokens)` - 토큰 포맷팅 (동일 로직)

## 📌 참고사항

1. **토큰 계산은 추정값**: 정확한 LLM 토큰 계산이 아닌 한글 3자 = 1토큰 기준 추정
2. **비동기 기록**: 방문 기록은 `@Async`로 비동기 처리하여 응답 지연 방지
3. **봇 필터링**: User-Agent 패턴으로 검색엔진 봇 제외 (googlebot, bingbot 등)
4. **캐싱 전략**: Summary 데이터는 5분간 캐싱되어 성능 최적화
5. **클라이언트 해시**: 세션 기반 또는 IP+User-Agent 해시로 고유 방문자 식별
6. **에러 처리**: 통계 조회 실패 시 빈 데이터 반환하여 UI 오류 방지
7. **CLAUDE.md 컨벤션 준수**: `@ManyToOne`만 사용, Boolean 필드 `is` 접두사
