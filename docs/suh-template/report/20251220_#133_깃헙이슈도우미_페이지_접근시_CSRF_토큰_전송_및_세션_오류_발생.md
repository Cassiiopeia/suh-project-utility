### 📌 작업 개요
프로덕션 환경에서 `/issue-helper` 페이지 접근 시 CSRF 토큰 전송 및 세션 오류로 인한 302 리다이렉트 문제 수정

**보고서 파일**: `.report/20251220_#133_깃헙이슈도우미_페이지_접근시_CSRF_토큰_전송_및_세션_오류_발생.md`

### 🔍 문제 분석
프로덕션 환경에서 `/issue-helper` 페이지 접근 시 다음 문제 발생:
- 페이지는 접근 가능하지만 API 호출 시 302 리다이렉트 발생
- 버전 정보 API(`/api/module/get/versions`) 호출 시 로그인 페이지 HTML 반환
- 브랜치/커밋 메시지 생성 API(`/api/issue-helper/create/commit-branch`) 호출 시 302 오류 발생
- 로컬 환경에서는 정상 동작

**근본 원인**:
1. `/issue-helper` 페이지가 `PublicEndpointConfig`에 등록되어 있지 않아 인증 필요 상태였음
2. `GlobalModelController`에서 `request.getSession(false)` 사용으로 세션이 없으면 세션을 생성하지 않음
3. 세션이 없으면 Spring Security가 CSRF 토큰을 생성하지 않음
4. AJAX 요청 시 CSRF 토큰이 없거나 빈 값으로 전송되어 서버에서 CSRF 검증 실패
5. 인증이 필요한 API는 세션 만료로 인해 302 리다이렉트 발생

### ✅ 구현 내용

#### `/issue-helper` 페이지를 공개 엔드포인트에 추가
- **파일**: `Suh-Web/src/main/java/me/suhsaechan/web/config/PublicEndpointConfig.java`
- **변경 내용**: `/issue-helper`를 `getPublicResourceEndpoints()` 목록에 추가
- **이유**: `/profile` 페이지에서 접근 가능해야 하므로 인증 없이 접근 가능하도록 설정

```java
// 변경 전
"/profile",
"/translator",    // AI 번역기 페이지

// 변경 후
"/profile",
"/issue-helper",  // GitHub 이슈 도우미 페이지
"/translator",    // AI 번역기 페이지
```

#### CSRF 토큰 생성을 위한 세션 생성 보장
- **파일**: `Suh-Web/src/main/java/me/suhsaechan/web/controller/view/GlobalModelController.java`
- **변경 내용**: `request.getSession(false)`를 `request.getSession(true)`로 변경
- **이유**: 공개 페이지에서도 세션이 생성되도록 하여 CSRF 토큰이 정상적으로 생성되도록 보장

```java
// 변경 전
HttpSession session = request.getSession(false);
if (session != null) {
  String clientHash = (String) session.getAttribute(ServerInfo.CLIENT_HASH_SESSION_KEY);
  model.addAttribute("clientHash", clientHash != null ? clientHash : "");
}

// 변경 후
HttpSession session = request.getSession(true);

String clientHash = (String) session.getAttribute(ServerInfo.CLIENT_HASH_SESSION_KEY);
model.addAttribute("clientHash", clientHash != null ? clientHash : "");
```

**특이사항**:
- `request.getSession(true)`는 세션이 없으면 자동으로 생성
- 모든 페이지에서 세션이 생성되므로 CSRF 토큰이 항상 생성됨
- 다른 공개 페이지(`/profile`, `/translator`)도 동일한 문제가 있었을 수 있지만, 이 변경으로 모두 해결됨

### 🔧 주요 변경사항 상세

#### PublicEndpointConfig.java
`/issue-helper` 경로를 공개 엔드포인트 목록에 추가하여 Spring Security의 `permitAll()` 처리되도록 설정
이제 인증 없이 페이지 접근 가능하며, `/profile` 페이지에서 링크로 접근 가능

#### GlobalModelController.java
`@ModelAttribute` 메서드에서 세션 생성 로직 변경
- 변경 전: 기존 세션이 없으면 세션을 생성하지 않음 (`getSession(false)`)
- 변경 후: 세션이 없으면 자동으로 생성 (`getSession(true)`)

**영향 범위**:
- 모든 페이지에서 세션이 생성됨
- CSRF 토큰이 항상 생성되어 AJAX 요청 시 정상적으로 전송됨
- 공개 페이지에서도 인증이 필요한 API 호출 시 CSRF 토큰 문제 해결

### 🧪 테스트 및 검증
- 인증되지 않은 상태에서 `/issue-helper` 페이지 접근 확인
- 페이지 로드 시 CSRF 토큰이 메타 태그에 포함되는지 확인
- 버전 정보 API(`/api/module/get/versions`) 호출 시 정상 응답 확인
- 브랜치/커밋 메시지 생성 API(`/api/issue-helper/create/commit-branch`) 호출 시 정상 동작 확인
- 프로덕션 환경에서 실제 테스트 필요

- 다른 공개 페이지(`/profile`, `/translator`)도 동일한 문제가 있었을 수 있지만, `GlobalModelController` 변경으로 모두 해결됨
- 번역 사이트(`/translator`)도 동일한 이슈가 있었을 수 있지만, 이미 `PublicEndpointConfig`에 등록되어 있고 `GlobalModelController` 변경으로 해결됨
