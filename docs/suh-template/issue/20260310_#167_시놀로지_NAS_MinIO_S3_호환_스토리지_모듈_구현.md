⚙️[기능추가][Storage][MinIO] 시놀로지 NAS MinIO S3 호환 스토리지 모듈 구현

📝 현재 문제점
---

- 현재 파일 저장은 SMB/SSH 방식으로 시놀로지 NAS에 접근하고 있으며, S3 표준 API를 지원하지 않음
- 외부 공개 파일은 `/volume1/web/` 경로에만 저장 가능한 제약이 있음
- 파일 접근 제어(공개/비공개), 임시 공유 링크(Presigned URL), 버킷 단위 관리 등 S3 표준 기능 부재
- 다른 프로젝트에서 동일한 스토리지를 재사용하려면 SMB 설정을 각각 해야 함

🛠️ 해결 방안 / 제안 기능
---

- 시놀로지 NAS Docker에 **MinIO 컨테이너** 설치 (S3 호환 오픈소스 스토리지 서버)
- suh-project-utility에 **Suh-Domain-Storage** 도메인 모듈 추가
- AWS S3 SDK 또는 MinIO SDK를 통해 MinIO에 접근하는 스토리지 서비스 구현
- UI에서 버킷(프로젝트) 생성/관리, 파일 업로드/다운로드 기능 제공

⚙️ 작업 내용
---

- MinIO Docker Compose 설정 및 시놀로지 NAS 배포
- Suh-Domain-Storage 모듈 생성 (Entity, Repository, Service, DTO)
- MinIO 연결 Configuration 설정 (endpoint, access-key, secret-key)
- 버킷 CRUD API 구현
- 파일 업로드/다운로드/삭제 API 구현
- Presigned URL 생성 기능 구현
- 스토리지 관리 UI 페이지 구현
