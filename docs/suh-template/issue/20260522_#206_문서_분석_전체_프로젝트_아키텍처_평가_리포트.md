📝 현재 문제점
---

전체 프로젝트(13개 멀티모듈 Spring Boot)에 대한 통합 분석이 누락돼 있어 아래 사항이 가시화되지 않은 상태다.

- CLAUDE.md 절대 규칙(예: `@OneToMany` 금지, 응답 DTO 금지 필드) 위반 케이스가 코드베이스에 산재
- Application 레이어/Web 레이어 책임 경계 모호 — 도메인 모듈에서 Web 모듈로의 역방향 테스트 의존
- 컨벤션 자동 강제 메커니즘(ArchUnit/Checkstyle 등) 부재로 위반 휴먼 감지에만 의존
- `@Value` 36건 / `ServerOptionKey` 5건 — 운영 중 변경 가능한 값이 재배포 필요한 형태로 박혀 있음
- Optional 패턴 미준수(`== null` 비교)가 54개 파일로 광범위
- Suh-Domain-Study 모듈 디렉터리 잔존(이슈 #176 제거 작업 후 cleanup 미완) 의심

전체 아키텍처를 객관적으로 평가하고 개선 우선순위를 명문화한 리포트가 필요하다.

🛠️ 해결 방안 / 제안 기능
---

`/cassiiopeia:analyze` 결과를 바탕으로 한 **아키텍처 평가 + 개선포인트 리포트**를 작성한다.

**리포트 포함 항목**:
- 모듈 구조 매트릭스 (의존성/외부라이브 위반/테스트 의존)
- CLAUDE.md 컨벤션 준수 카운트
- P0(절대 규칙 위반) / P1(아키텍처 결함) / P2(개선 권고) 분류
- 우선순위별 액션 플랜 (Phase 1~4)
- 영역별 점수(7개 영역) + 종합 점수

**산출물**:
- `docs/suh-template/report/YYYYMMDD_{이슈번호}_아키텍처_평가_리포트.md`

**본 이슈 범위**: 리포트 작성까지. 실제 개선 작업은 후속 이슈로 분리한다(P0 우선 처리 → 별도 이슈 발행 예정).
