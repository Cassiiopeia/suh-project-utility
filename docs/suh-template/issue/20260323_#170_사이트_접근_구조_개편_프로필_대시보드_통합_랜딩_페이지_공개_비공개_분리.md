🚀[기능개선][사이트구조][Security][UI] 사이트 접근 구조 개편 : 프로필+대시보드 통합 랜딩 페이지 + 공개/비공개 분리

📝 현재 문제점
---

- 랜딩(`/`)이 dashboard로 연결되는데 로그인이 필요하여 외부 방문자가 사이트에 진입할 수 없음
- profile.html과 dashboard.html에 기능 카드가 중복되어 새 기능 추가 시 두 곳을 동시에 수정해야 함 (관리포인트 2배)
- Study 기능이 불필요하지만 코드(Entity, Repository, Service, Controller, HTML)가 남아있음
- "나를 자랑하는 사이트"로서 포트폴리오/쇼케이스 역할을 제대로 못하고 있음

🛠️ 해결 방안 / 제안 기능
---

- 랜딩 페이지(`/`)를 프로필 + 대시보드 + 기능 쇼케이스를 통합한 단일 페이지로 재구성
- 대부분의 페이지를 공개하고, 관리/운영 기능만 로그인 필요로 변경
- 기능 카드 목록을 Thymeleaf fragment로 분리하여 한 곳에서만 관리
- Study 모듈 관련 코드 제거 (파일 업로드 기능은 유지)

⚙️ 작업 내용
---

### 1. 랜딩 페이지(`/`) 통합 재구성
- 프로필 헤더 (사진, 이름, 소셜링크)
- About Me
- Timeline + Certifications
- Tech Stack
- Team Projects + Personal Projects + Nexus Modules
- 서버 통계 (방문자, 챗봇, 토큰 사용량 등 - 공개)
- 공지사항
- Dev Tools 기능 카드 쇼케이스
  - 공개 기능: 바로 이동 가능
  - 비공개 기능: 잠금 아이콘 + "로그인 필요" 표시
- Task D-Day (공개)
- Contact

### 2. 접근 권한 분리
- **공개 페이지**: `/`, `/issue-helper`, `/translator`, `/sejong-auth`, `/suh-random`, `/suh-devops-template`, `/openai-chat`
- **로그인 필요 페이지** (7개만): `/docker-logs`, `/ai-server`, `/somansa/bus`, `/chatbot-management`, `/notice-management`, `/grass`, `/task-tracker`

### 3. 관리포인트 최적화
- 기능 카드 목록을 Thymeleaf fragment로 분리하여 한 곳에서만 관리

### 4. 제거 대상
- Study 모듈 (Entity, Repository, Service, Controller, HTML)
- 기존 `/profile` 페이지 (랜딩에 통합)
- 기존 `/dashboard` 페이지 (랜딩에 통합)
- 파일 업로드 기능은 유지

### 5. Security 변경
- PublicEndpointConfig 수정 (대부분 공개, 7개 페이지만 인증 필요)
- 로그인 성공 후 리다이렉트 대상을 `/dashboard` → `/`로 변경
