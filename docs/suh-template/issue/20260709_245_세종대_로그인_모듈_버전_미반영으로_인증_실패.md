🗒️ 설명
---

세종대 인증 기능(`/api/sejong-auth`)이 모든 인증 방식(INTEGRATED / DHC / SIS / RAW)에서 `CONNECTION_FAILED`로 실패한다.

근본 원인은 이 프로젝트가 참조하는 세종대 로그인 라이브러리 `kr.suhsaechan:sejong-univ-auth` 버전이 구버전 `1.2.0`에 고정되어 있기 때문이다. 라이브러리 측에서 세종대 포털의 HTTP 평문 서비스 중단(`classic.sejong.ac.kr`)에 대응해 SSO 리다이렉트 URL을 `http` → `https`로 전환하는 수정을 완료했고, 해당 수정은 `1.2.5` / `1.2.6`에 포함되어 Nexus(`maven-releases`)에 이미 배포되었다. 그러나 이 프로젝트는 여전히 수정 이전 버전인 `1.2.0`을 사용하고 있어 반영되지 않았다.

- 관련 파일: `Suh-Common/build.gradle` (`api 'kr.suhsaechan:sejong-univ-auth:1.2.0'`)
- 라이브러리 원인 이슈: Cassiiopeia/SUH-sejong-univ-auth#10 (SSO 리다이렉트 URL http→https 전환, 1.2.5/1.2.6 반영·배포 완료)

🔄 재현 방법
---

1. 세종대 인증 페이지(`/sejong-auth`)로 이동
2. 학번/비밀번호를 입력하고 인증 버튼 클릭
3. 정상 계정임에도 `CONNECTION_FAILED` 에러로 인증 실패

📸 참고 자료
---

- 에러 코드: `CONNECTION_FAILED` (세종대 포털 연결 실패)
- 라이브러리 변경: `1.2.0` → `1.2.6` (API 시그니처 변경 없음, breaking change 없음)

✅ 예상 동작
---

- 정상 학번/비밀번호로 인증 시 `CONNECTION_FAILED` 없이 학생 정보(학과·이름·학년 등)가 정상 반환되어야 함
- `Suh-Common/build.gradle`의 의존성을 `sejong-univ-auth:1.2.6`으로 올려 포털 https 전환 대응이 반영되면 해결

- **대상 기능**: 세종대 인증 (`/api/sejong-auth`)
