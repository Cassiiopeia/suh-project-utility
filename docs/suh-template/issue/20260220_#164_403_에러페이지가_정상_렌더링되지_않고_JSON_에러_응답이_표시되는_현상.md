❗[버그][에러페이지][Security] 403 에러 페이지가 정상 렌더링되지 않고 JSON 에러 응답이 표시되는 현상

🗒️ 설명
---

- 권한이 없는 사용자가 관리자 전용 페이지(`/notice-management`)에 접근할 때, `error/403.html` Thymeleaf 템플릿이 렌더링되지 않고 JSON 에러 응답이 표시됨
- 직접 `/error/403` URL로 접근해도 동일하게 JSON 에러 발생
- 에러 응답 내용:
```json
{
  "errorCode": "INTERNAL_SERVER_ERROR",
  "httpStatus": "INTERNAL_SERVER_ERROR",
  "message": "No static resource error/403."
}
```

🔄 재현 방법
---

1. 일반 사용자(비관리자) 계정으로 로그인
2. `https://lab.suhsaechan.kr/notice-management` 접근
3. 403 에러 페이지 대신 JSON 에러 응답이 표시됨

📸 참고 자료
---

- 에러 발생 URL: `https://lab.suhsaechan.kr/error/403`

✅ 예상 동작
---

- 권한이 없는 사용자가 관리자 페이지에 접근하면 `templates/error/403.html`이 렌더링되어야 함
- 잠금 아이콘 + "접근 권한이 없습니다" 메시지 + 홈/로그인 버튼이 표시되어야 함

🔍 원인 분석
---

1. **PageController의 잘못된 리다이렉트**: `return "redirect:/error/403"` 사용 → `/error/403` URL에 매핑된 핸들러가 없어서 404 발생 → GlobalExceptionHandler가 JSON으로 응답
2. **CustomErrorController 매핑 범위**: `/error` 경로만 매핑되어 있고 `/error/403` 같은 서브경로는 미매핑
3. **GlobalExceptionHandler가 모든 예외를 JSON으로 처리**: `@RestControllerAdvice`로 페이지 요청의 예외도 JSON으로 반환
4. **Spring Security exceptionHandling 미설정**: Security 레벨의 403(AccessDenied) 처리 핸들러 없음
5. **PublicEndpointConfig에 `/error/**` 경로 누락**: 미인증 사용자가 에러 페이지에 접근 불가

⚙️ 작업 내용
---

- [ ] PageController: `redirect:/error/403` → 직접 템플릿 반환(`error/403`) + HTTP 403 상태코드 설정
- [ ] WebSecurityConfig: `exceptionHandling` + `accessDeniedHandler` 추가
- [ ] PublicEndpointConfig: `/error/**` 공개 경로 추가
