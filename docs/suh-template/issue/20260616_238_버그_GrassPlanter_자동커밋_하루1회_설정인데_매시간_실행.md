🗒️ 설명
---

GrassPlanter(GitHub 잔디 자동 커밋) 기능이 화면상 **"하루 1회 / 목표 레벨 Lv.1"** 로 설정되어 있는데도, 실제로는 **매시간 정각마다** 자동 커밋이 실행된다.

- 커밋 로그에 매시간 1줄씩(00:00, 01:00, 02:00 …) 자동 커밋 기록이 쌓인다.
- 모든 로그의 기여 레벨(commitLevel)이 계속 `0`으로 찍힌다.

원인은 다음과 같다.

1. 자동 커밋 스케줄러의 cron이 `0 0 * * * *`(매시간 정각)으로 하드코딩되어 있다. 화면의 "하루 1회" / "목표 레벨" 설정은 스케줄러가 전혀 참조하지 않는다(빈도 설정용 엔티티는 정의만 되어 있고 실제로 사용되지 않음).
2. 매시간 깨어나 "오늘 기여 레벨 < 목표 레벨"이면 커밋하는 구조인데, 오늘 기여 여부를 GitHub 공개 프로필 페이지 HTML 스크래핑으로 확인한다. 방금 한 커밋이 즉시 반영되지 않아 매시간 "아직 목표 미달"로 판단 → 매시간 중복 커밋된다.
3. 기존 스크래핑 방식은 private 기여를 누락하고, GitHub HTML 구조가 바뀌면 항상 0으로 떨어져 매시간 커밋되는 취약점도 있다.

🔄 재현 방법
---

1. GrassPlanter 페이지(`/grass`)에서 프로필을 "자동 커밋 활성화", "하루 1회"로 설정한다.
2. 하루 동안 방치한다.
3. 커밋 로그를 확인하면 하루 1회가 아니라 매시간 1건씩 자동 커밋이 쌓여 있고, 모든 기록의 레벨이 0으로 표시된다.

📸 참고 자료
---

커밋 로그 예시 (매시간 반복, 레벨 0 고정):

```
2026. 6. 16. 오후 2:00:03    Cassiiopeia    Cassiiopeia/auto-commit    Auto-commit: Daily contribution    자동    성공    0
2026. 6. 16. 오후 1:00:02    Cassiiopeia    Cassiiopeia/auto-commit    Auto-commit: Daily contribution    자동    성공    0
2026. 6. 16. 오후 12:00:02   Cassiiopeia    Cassiiopeia/auto-commit    Auto-commit: Daily contribution    자동    성공    0
...
```

✅ 예상 동작
---

- 자동 커밋은 **매일 23시(KST)에 1회만** 실행되어야 한다.
- 그 시각에 오늘 GitHub 기여(잔디)가 **이미 있으면 자동 커밋을 생략**하고, **없을 때만 1번 커밋**해 잔디 빵꾸만 메워야 한다.
- 사용자가 낮에 직접 커밋해 잔디가 이미 심어졌다면 자동 커밋이 발생하지 않아야 한다.
- 오늘 기여 여부 판정은 private 기여까지 포함하는 정확한 방식이어야 한다.
