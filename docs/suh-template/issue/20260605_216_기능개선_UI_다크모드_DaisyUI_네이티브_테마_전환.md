📝 현재 문제점
---

전 페이지에서 다크모드가 일관되게 동작하지 않는다. 다크모드를 켜도 일부 영역(특히 `bg-white`, `text-gray-*` 등 Tailwind 하드코딩 색상이 쓰인 곳)은 흰 배경/검은 글씨로 남아, 다크 배경 위에서 이상하게 보인다.

근본 원인은 다크모드를 **DaisyUI 네이티브 테마 시스템으로 구현하지 않은 것**이다.

- `fragments/header.html`이 Tailwind를 `cdn.tailwindcss.com`(Tailwind v3 런타임)으로 로드한다.
- 그러나 함께 로드하는 DaisyUI 5(`daisyui@5.4.2/daisyui.css`)는 Tailwind v4의 토큰 시스템(`@theme`/`@plugin`)을 전제로 만들어졌다.
- 비호환 조합이라 DaisyUI의 색/테마 토큰이 제대로 깔리지 않는다. 그 결과 `base-100` / `base-content` 같은 시맨틱 색이 테마에 따라 자동으로 바뀌지 않는다.
- 이를 가리기 위해 `common.css`에서 컴포넌트·유틸리티마다 `[data-theme="dark"] .xxx { ... !important }` 형태로 다크 색을 일일이 수동 오버라이드해 왔다. 현재 `common.css`는 약 3,700줄이며 다수가 이 수동 오버라이드다.
- 동시에 페이지 HTML에는 `bg-white`, `text-gray-700` 같은 하드코딩 색상이 광범위하게 흩어져 있다(16개 페이지에 걸쳐 약 471곳). 이 중 `common.css`에서 미처 덮지 못한 곳이 다크모드에서 깨져 보인다.

즉 "새 마크업을 추가할 때마다 다크모드가 깨지고, 그때마다 CSS에서 `!important`로 막는" 악순환 구조다. 유지보수가 사실상 불가능하다.

참고 이력:
- 직전에 `@tailwindcss/browser@4`(브라우저 런타임 CDN) + `@plugin "daisyui"` 방식으로 v4 전환을 시도했으나 revert됨(커밋 5a50a36 → revert ee8b7bf). 당시 설계 문서에 런타임 컴파일로 인한 초기 렌더 지연/FOUC, 내부망 검증 불가가 리스크로 기록돼 있다.
- 현재는 revert 후 `common.css`에 버튼 색 룰만 추가하는 최소 변경으로 색만 임시 복구한 상태다(커밋 8e2e35c).

관련 파일:
- `Suh-Web/src/main/resources/templates/fragments/header.html` (Tailwind/DaisyUI CDN 로드)
- `Suh-Web/src/main/resources/static/css/common.css` (다크모드 수동 오버라이드 집약)
- `Suh-Web/src/main/resources/static/js/common.js` (`initTheme` / 테마 토글)
- `Suh-Web/src/main/resources/templates/pages/*.html` (하드코딩 색상 다수)

🛠️ 해결 방안 / 제안 기능
---

다크모드를 DaisyUI 네이티브 테마 시스템으로 전환해, 테마(라이트/다크)에 따라 시맨틱 색이 자동으로 적용되도록 한다.

- DaisyUI 5가 정상 동작하도록 Tailwind 셋업을 정합 조합으로 맞춘다(네이티브 테마 토큰이 실제로 적용되는 환경 구성).
- 페이지의 하드코딩 색상(`bg-white`, `text-gray-*` 등)을 시맨틱 토큰(`bg-base-100`, `text-base-content` 등)으로 점진 치환한다.
- 전환이 완료된 범위부터 `common.css`의 수동 `[data-theme="dark"]` 오버라이드를 제거해, 테마 시스템 단일 출처로 색을 관리한다.
- 내부망(외부 패키지 다운로드 불가) 제약과 직전 v4 시도의 revert 사유(런타임 컴파일 성능/FOUC, 검증 불가)를 고려해, 어떤 셋업 방식으로 갈지 설계 단계에서 확정한다.
