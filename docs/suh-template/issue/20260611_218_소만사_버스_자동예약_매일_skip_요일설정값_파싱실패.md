🗒️ 설명
---

소만사 버스 자동예약을 월~금으로 켜놓았으나 매일 아침 확인하면 예약이 되어 있지 않다. 스케줄러가 매일 "비허용 요일이라 예약 skip"으로 판정하고 넘어간다.

**근본 원인**

`SomansaBusSchedulerService.parseDays()`가 요일 설정값을 `DayOfWeek.valueOf()`로 직접 파싱한다.

- 설정값(`ServerOptionKey.SOMANSA_BUS_SCHEDULER_DAYS`)의 기본값과 설명이 약어 기준(`MON,TUE,WED,THU,FRI`)이다.
- 그러나 Java `DayOfWeek` enum은 `MONDAY`, `TUESDAY`… 정식 명칭만 받는다.
- 따라서 `DayOfWeek.valueOf("MON")`이 매번 `IllegalArgumentException`을 던져, 월~금 전부 "알 수 없는 요일 설정값"으로 무시된다.
- 결과적으로 허용 요일 Set이 비어버려, 모든 날짜가 "비허용 요일"로 판정되고 예약이 매일 skip된다.

즉 설정을 올바르게(`MON,TUE,...`) 넣어도 코드가 약어를 해석하지 못해 자동예약이 100% 무력화된다.

🔄 재현 방법
---

1. 자동예약 실행 요일을 `MON,TUE,WED,THU,FRI`로 설정 (기본값과 동일).
2. 스케줄러 실행 시각까지 대기.
3. 로그에서 아래 흐름 확인.

```
WARN  SomansaBusSchedulerService : 알 수 없는 요일 설정값 무시: MON
WARN  SomansaBusSchedulerService : 알 수 없는 요일 설정값 무시: TUE
WARN  SomansaBusSchedulerService : 알 수 없는 요일 설정값 무시: WED
WARN  SomansaBusSchedulerService : 알 수 없는 요일 설정값 무시: THU
WARN  SomansaBusSchedulerService : 알 수 없는 요일 설정값 무시: FRI
ERROR SomansaBusSchedulerService : 허용 요일 설정이 비어있거나 잘못됨 — fallback: 다음날 사용
INFO  SomansaBusSchedulerService : 내일(WEDNESDAY) 비허용 요일 — 예약 skip, nextFireAt 만 갱신
```

4. 예약이 수행되지 않고 skip됨.

📸 참고 자료
---

- 발생 위치: `Suh-Domain-Somansa-Bus/.../service/SomansaBusSchedulerService.java` `parseDays()`
- 설정 키: `Suh-Common/.../constant/ServerOptionKey.java` `SOMANSA_BUS_SCHEDULER_DAYS` (기본값 `MON,TUE,WED,THU,FRI`)
- 로그 출처: `suh-project-utility-green` 컨테이너

> 참고: 인증서/SSL(PKIX) 문제가 아니다. 별개 시스템이던 `auto-bus-reservation` 레포와도 무관하며, 현재 운영 중인 본 레포(`Suh-Domain-Somansa-Bus`)의 요일 파싱 버그다.

✅ 예상 동작
---

- 설정값 `MON,TUE,WED,THU,FRI`(약어) 또는 `MONDAY,...`(정식명) 모두 정상 파싱되어야 한다.
- 허용 요일에 해당하는 날에는 자동예약이 정상 수행되어야 한다.
- 월~금 설정 시 평일마다 예약이 성공해야 한다.
