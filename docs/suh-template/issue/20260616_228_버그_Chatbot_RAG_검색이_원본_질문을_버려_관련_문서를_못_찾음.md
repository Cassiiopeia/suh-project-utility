🗒️ 설명
---

RAG 챗봇이 관련 문서가 분명히 등록되어 있는데도 "관련 문서 없음"으로 답하며 제대로 검색하지 못한다.

- admin 챗봇 문서 관리에 문서 9개·청크 49개가 정상 등록되어 있으나, "너는 누구야?" 같은 질문에서 `서새찬 개발자` 등 명백히 관련된 문서를 검색하지 못한다.
- 근본 원인: **의도 분류 단계가 사용자의 원본 질문을 버리고 인공 키워드 쿼리로 치환**해서 검색한다. `ChatbotService.classifyUserIntentInternal`의 프롬프트(`searchQuery 생성 규칙`)가 LLM에게 "반드시 'SUH Project Utility' 키워드 포함" 형태의 쿼리를 만들도록 강제하고, `getSearchQuery`가 그 LLM 생성 쿼리를 원본 질문보다 우선해서 사용한다.
- 그 결과 실제 문서 청크의 문장들과 임베딩 유사도가 멀어져 검색이 빈손으로 돌아온다.
- 부가 문제: RAG 검색 "결과 0건"이 `minScore` 필터에서 전부 걸러진 것인지, Qdrant 원본부터 0건인지 현재 로그로 구분이 불가능해 원인 진단이 막혀 있다.
- 부가 문제: `minScore`/`topK`가 코드에 하드코딩되어 있어 운영 중 admin UI에서 조정할 수 없다.

🔄 재현 방법
---

1. 챗봇 페이지로 이동
2. "너는 누구야?" 입력
3. 의도 분류는 `KNOWLEDGE_QUERY (95%)`로 검색을 시도하지만, 실제 검색 쿼리가 `SUH Project Utility 소개 목적 주요 기능 모듈`로 치환됨
4. "관련 문서 없음" 표시 → `서새찬 개발자` 문서가 있음에도 검색 실패
