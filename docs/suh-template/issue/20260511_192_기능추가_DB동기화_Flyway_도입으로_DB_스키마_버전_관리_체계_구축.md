# ⚙️[기능추가][DB동기화] Flyway 도입으로 DB 스키마 버전 관리 체계 구축

- 라벨: 작업전
- 담당자: Cassiiopeia

---

📝 현재 문제점
---

- `ddl-auto: update`만 사용하여 스키마 변경 이력 추적 불가
- `somansa_bus_route` 테이블에 `is_shuttle` 컬럼이 엔티티에 추가됐으나 기존 DB에 미반영되어 런타임 에러 발생
- DB 완전 초기 상태(테이블 없음)에서 jar 실행 시 Hibernate DDL만으로는 안전하게 처리 불가

🛠️ 해결 방안 / 제안 기능
---

- Flyway 도입: `ddl-auto: update` 유지 + Flyway 병행 운영 (Flyway가 DDL보다 먼저 실행되어 선제 적용)
- 마이그레이션 파일 네이밍: `version.yml` 버전 기준 `V{major}_{minor}_{patch}__{설명}.sql`
- 완전 초기화 안전 전략: `CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ADD COLUMN IF NOT EXISTS` 패턴으로 초기/기존 DB 모두 커버

⚙️ 작업 내용
---

- `Suh-Common/build.gradle` — `flyway-core`, `flyway-database-postgresql` 의존성 추가
- `application.yml` — `spring.flyway` 설정 추가 (`baseline-on-migrate: true`, `validate-on-migrate: false`)
- `Suh-Web/src/main/resources/db/migration/V2_5_27__add_is_shuttle_to_somansa_bus_route.sql` — `somansa_bus_route` 초기 스키마 + `is_shuttle` 컬럼 추가 마이그레이션
- `CLAUDE.md` — Flyway 마이그레이션 규칙 및 초기화 안전 전략 문서화
